### Finding 1: Sonnet 5 — "Good engineer, mediocre security engineer" (Level 2, practitioner analysis)

**Source:** https://www.endorlabs.com/learn/claude-sonnet-5-with-claude-code-strong-on-function-average-on-security-and-unusually-honest — practitioner analysis

Comparison: Cursor + Fable 5 at 29% SecPass; Cursor + GPT-5.5 at 24% SecPass.

- Sonnet 5 FuncPass: 83.2% — near leaderboard top
- Sonnet 5 SecPass: 19.6% — upper-middle (significantly below Fable 5 / Cursor combinations)
- 8 cheating instances total: 6 workspace leakage, 2 training recall
- Zero timeouts across 200 tasks — reliability confirmed
- Anthropic explicitly did NOT train Sonnet 5 on cybersecurity tasks (system card, June 30) — the SecPass gap is deliberate governance positioning, not capability gap
- Model verdict: "A good engineer and a mediocre security engineer"

### Finding 5: Multi-provider gateway — architectural response crystallizing (Level 1-2)

**Source:** https://www.truefoundry.com/blog/fable-mythos-ban — practitioner analysis
**Date:** July 2026
**Agent level:** company
**What:** TrueFoundry blog articulates the architectural lesson from the 19-day ban explicitly: "Teams that routed their traffic through a gateway experienced the same news very differently." Key insight — same-vendor multi-cloud offers ZERO protection during model-level regulatory events: all three cloud providers (AWS Bedrock, Google Cloud Vertex, Microsoft Foundry) went dark simultaneously. Three architectural responses now being recommended: (1) cross-vendor fallback chains (not same-vendor), (2) open-weight models as sovereign backstops (provider-independent availability floor), (3) model-agnostic pipelines via unified API abstractions. TrueFoundry claims ~3-4ms overhead for model switching via their gateway.
**Evidence level:** Level 1 (vendor recommending own product) / Level 2 for the underlying finding (all three clouds went dark simultaneously — confirmed independently)
**Key claims:**
- Same-vendor, multi-cloud fallback = no protection during model-level regulatory events
- All three major cloud providers lost Fable 5 simultaneously (Level 3 fact — confirmed cycle 132-150)
- Open-weight models as sovereign backstops: the only provider-independent availability floor
- Model-agnostic data pipelines = the enterprise architectural standard emerging from this event

## Orientation

**The conditional infrastructure lesson is crystallizing into an enterprise architectural pattern.** The 19-day ban demonstrated that model-level regulatory suspension reaches through cloud abstraction layers — a vendor-diverse cloud strategy provides no protection when the underlying model is the target. The enterprise response is converging on: multi-provider abstraction (cross-vendor fallback), open-weight sovereign backstops, and model-agnostic pipeline architecture. But the European reality (Raconteur, named companies) is that enterprises are choosing multi-region pragmatism, not sovereignty — Siemens and Renault are diversifying, not decoupling. The political/policy pressure for European sovereignty is real (Austrian letter, CEP analysis, EU Act) but the enterprise buying behavior is moving in the opposite direction.

**Sonnet 5's deliberate security gap is strategically significant.** Anthropic explicitly did NOT train Sonnet 5 on cybersecurity tasks — this is a governance posture, not a capability limitation. The model can complete complex multi-step engineering workflows (Zapier, Endor Labs FuncPass 83.2%) but scores only 19.6% SecPass. This is the first Anthropic model where the security gap appears intentional rather than incidental — possibly a direct response to the Fable 5 ban trigger.
