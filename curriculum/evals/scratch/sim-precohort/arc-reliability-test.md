# Arc note: the test-learn-encode loop across AE101 M4-M6

## The throughline

One pattern runs the whole body of work, and it is genuinely present rather than imposed: **the same scoped task is run, then re-run, and the contrast between runs is treated as the curriculum.** Everything is built so that two runs of one task produce the evidence the student reasons from. The M4 module sends the task off un-packaged and on purpose: "Send off the task un-packaged in the same Claude Code session, no plan.md, no verifier, no reference artifact, and let it run." M5 re-sends the identical task with packaging as the only changed variable, and names that explicitly: "Same task run twice with packaging as the only changed variable IS the lesson." M6 then closes the loop by packaging the diagnosis into a reusable move: "The second authored skill IS the loop." The lecture trio gives the loop its vocabulary -- failure modes (reading-the-return), the name "eval" (the-loop-has-a-name), and the flywheel (agents-that-build-agents).

## Shape at the start (M4) vs the end (M6)

At the start the work is deliberately raw and single-shot. M4's whole posture is "run it and see" -- the artefact is whatever comes back, including a stalled run. reading-the-return plants the one diagnostic question that turns that raw artefact into readable data: "What would have caught this earlier in the run?" Nothing is packaged yet; nothing is named yet. The student has a trace and a question.

By the end the work is reflexive and accretive. M6 isn't reading a run, it's reading two runs against each other, routing each gap to a home (memory, verifier, or skill), and shipping a skill so the lesson survives. the-loop-has-a-name supplies the noun that lets it compose: "Your verifier is an eval. Your judge is an eval. Your gate is an eval." agents-that-build-agents extends the same primitive one turn further -- the agent proposing the next scaffold. The shift is from *produce an artefact* to *produce a thing that improves the next artefact*.

## Where the arc is uneven

It is not perfectly even. The test-learn-encode triad is fully load-bearing in M5 and M6; in M4 the "encode" leg is intentionally absent ("No Debrief... Compound on evidence, not anticipation"), so M4 is the half-built setup, not a full turn of the loop. The naming also lags the doing on purpose: the word "eval" and "Ronacher's three-pattern" are withheld until the closing lectures, so a reader scanning M4-M5 sees the mechanism long before its label. The arc is real, but its first beat is a primer and its vocabulary arrives last by design.
