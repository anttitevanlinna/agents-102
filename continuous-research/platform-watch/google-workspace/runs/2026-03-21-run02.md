## Findings

### Jules: Asynchronous coding agent out of beta, powered by Gemini 2.5
**Source:** https://nelsonslog.wordpress.com/2026/03/18/first-impressions-of-jules-googles-coding-agent/ — [practitioner direct]
**Date:** 2026-03-18
**What:** Nelson's first-impressions review of Jules describes it as "instructing a junior engineer" — it found a duplicate entry the author had missed, but also "sometimes says it did things that it didn't." Jules is fully asynchronous (submit task, get PR back in ~10 minutes), uses GitHub PRs as review mechanism. The author preferred this workflow over Claude Code's text interface for certain tasks. Free tier: 15 tasks/day. Paid: $19.99/mo (AI Pro) or $124.99/mo (Ultra).
**Evidence level:** Level 2 (single practitioner experiment)

### Jules "Critic" feature: adversarial self-review before presenting code
**Source:** https://developers.googleblog.com/meet-jules-sharpest-critic-and-most-valuable-ally/ — [vendor press release]
**Date:** 2026-03 (approx)
**What:** Google added an internal critic to Jules that adversarially reviews every proposed code change before the user sees it. The critic flags problems and hands them back to Jules to fix. This is a real architectural feature (not marketing), but the only source is Google's own blog.
**Evidence level:** Level 0 (vendor blog — the feature exists, but no independent verification of its effectiveness)

### ADK 2.0 Alpha: graph-based workflows, TypeScript support, 7M+ downloads
**Source:** https://google.github.io/adk-docs/ — [practitioner direct / open-source repo]
**Additional:** https://developers.googleblog.com/introducing-agent-development-kit-for-typescript-build-ai-agents-with-the-power-of-a-code-first-approach/ — [vendor press release]
**Date:** 2026-Q1
**What:** Google's Agent Development Kit reached 7M+ Python downloads. ADK 2.0 Alpha introduces graph-based workflows (major orchestration upgrade). TypeScript SDK now available. ADK is open-source, model-agnostic (optimized for Gemini but works with others), and includes built-in eval (`adk eval`), CLI, and local web UI for debugging. Deploys to Vertex AI Agent Engine or anywhere via Docker/Cloud Run.
**Evidence level:** Level 2 (download numbers are real; developer experience reports come from multiple GDG events in Berlin, Paris)

### ADK vs OpenAI Agents SDK: ADK is heavier but more production-ready
**Source:** https://iamulya.one/posts/a-developer-guide-to-ai-agents-openai-agents-sdk-vs-google-adk/ — [practitioner analysis]
**Additional:** https://medium.com/@Micheal-Lanham/adk-typescript-vs-openai-agents-sdk-which-one-survives-retries-timeouts-and-human-approvals-808903d65a61 — [practitioner analysis]
**Date:** 2026-Q1
**What:** Head-to-head comparisons show ADK has steeper learning curve but more built-in production features: session state with pluggable backends, built-in eval, reflect-and-retry on tool failures (OpenAI SDK threw exceptions requiring external retry logic). OpenAI SDK is simpler, more opinionated, better for rapid prototyping. ADK better for Google Cloud-heavy environments and complex stateful applications. A2A protocol enables cross-framework agent communication.
**Evidence level:** Level 3 (multiple independent practitioners reaching same conclusions)

### Agentspace rebranded to Gemini Enterprise — GA, $25/user/month
**Source:** https://cloud.google.com/blog/products/ai-machine-learning/google-agentspace-enables-the-agent-driven-enterprise — [vendor press release]
**Additional:** https://cloudfresh.com/en/blog/google-agentspace-evolves-into-gemini-enterprise/ — [domain trade publication]
**Date:** 2025-Q4 to 2026-Q1
**What:** Agentspace has been rebranded and folded into "Gemini Enterprise" ($25/user/month). It's now GA — includes enterprise search across company data, agent creation/orchestration, Deep Research agent, NotebookLM integration. Security: HIPAA, FedRAMP High, VPC-SC, CMEK. Available on-premises via Google Distributed Cloud. A2A protocol v0.3 released.
**Evidence level:** Level 0 (vendor announcements only — no independent enterprise deployment case studies found)

### Gemini model stability problems: 503 errors, forced migrations, vanishing chat histories
**Source:** https://www.theregister.com/2026/02/23/gemini_users_say_their_chat/ — [general press]
**Additional:** https://help.apiyi.com/en/gemini-3-pro-deprecated-march9-migrate-3-1-pro-guide-en.html — [practitioner direct]
**Date:** 2026-02 to 2026-03
**What:** Gemini 3.1 Pro plagued by 503 errors and latencies up to 104 seconds since Feb 19 launch. Gemini 3 Pro shut down March 9 forcing migration. Multiple users report months of chat history vanishing after 3.1 rollout — Google acknowledged bug. Free tier quotas reduced without notice; post-Dec 2025 changes make free tier unsuitable for production. Rate limits significantly lower than OpenAI (150-300 RPM vs 500-10,000 RPM at Tier 1).
**Evidence level:** Level 3 (convergence — multiple independent reports of stability issues across forums, The Register, developer blogs)

### Vertex AI Agent Builder: full stack but Google Cloud lock-in
**Source:** https://venturebeat.com/ai/the-agent-builder-arms-race-continues-as-google-cloud-pushes-deeper-into — [general press]
**Additional:** https://docs.cloud.google.com/agent-builder/overview — [vendor documentation]
**Date:** 2025-11 to 2026-Q1
**What:** Vertex AI Agent Builder is the developer-facing platform: Agent Designer (visual low-code, Preview), Agent Engine (production scaling, GA), Agent Garden (pre-built sample agents). Tool governance via Cloud API Registry. Framework-agnostic deployment (ADK, LangGraph, etc.). However, deeply tied to Google Cloud services. Analyst Torsten Volk (Omdia) noted Google goes "deeper than some competitors" in developer experience but Bradley Shimmin (Futurum) warned Google "needs to build a developer ecosystem."
**Evidence level:** Level 1 (analyst opinions + vendor documentation, no independent practitioner production reports)

## What I Looked For But Did Not Find

- **Enterprise production case studies.** No named company reporting agentic deployment results on Google's platform with measurable outcomes. Every "enterprise" reference is either a vendor announcement or analyst prediction. This is the single biggest gap.
- **Practitioner convergence on Vertex AI Agent Builder.** Found plenty of vendor docs and tutorials, but no equivalent of the Codex practitioner review ecosystem. Developers are writing about ADK (the open-source framework), not about using Agent Builder in production.
- **Gemini Enterprise (Agentspace) real-world usage.** GA announced, pricing published, but zero independent reports of companies using it.
- **Jules vs Codex head-to-head comparisons.** Both are async coding agents — expected direct comparisons but found very few (most comparisons are Claude Code vs Codex).
- **Google agent security incidents beyond the CVE.** Found one CVE (bucket squatting) but no practitioner reports of agent-specific security issues in production.
- **Nordic companies using Google agent platform.** Zero findings.

## Orientation

Google has the broadest agent platform surface area — from coding (Jules) to enterprise search (Gemini Enterprise) to developer framework (ADK) to visual builder (Agent Designer) — but practitioner evidence is thinnest at the enterprise layer where it matters most. The open-source ADK is gaining real traction (7M downloads, active community) and compares favorably to OpenAI Agents SDK for production resilience, but Gemini model stability issues (503s, forced migrations, vanishing histories) undermine trust at exactly the moment Google needs developers to commit. The pattern: strong platform architecture, weak execution reliability.
