---
name: Module reusability across curriculums
description: Modules and exercises must be reusable building blocks — designed for composition across training variants, not locked to one curriculum
type: feedback
---

Build modules and exercises so they can be reused across different curriculum variants. Antti has variations in mind (e.g., "Agents 102 for mid-management" — more team/org transformation focus). ~80% of content should be shared across variants.

**Why:** The training is a platform, not a product. Different buyer segments need different emphasis (hands-on builders vs. mid-management transformation leaders) but the core competence-building is the same. Monolithic modules that assume one audience = rebuilding for every variant.

**How to apply:** Design each module as a self-contained composable unit. Separate the universal core (agent fundamentals, building, evals) from audience-specific framing (individual empowerment vs. team transformation vs. org strategy). Use plug points not just for org context but for audience context. When writing exercises, ask: "Does this work for a CTO building their first agent AND for a mid-manager thinking about team capability?" If not, split the audience-specific part into a variant layer.
