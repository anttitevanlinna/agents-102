### Finding 3: Cognitive Debt — AI-Native Team Failure Mode Named (Level 2, converging toward L3)

**Source A:** https://simonwillison.net/2026/jul/2/understand-to-participate/ — [practitioner direct]
**Date:** July 2, 2026
**Source B:** https://www.bvp.com/atlas/inside-ai-pilled-engineering-teams-five-lessons-for-scaling-without-losing-the-plot — [practitioner analysis]
**Date:** June 10, 2026
**Agent level:** team
**What:** Simon Willison (July 2) names *cognitive debt* as the primary AI-native team failure mode: "understanding drifts from how code actually works as AI builds increasingly large changes." You must maintain deep code understanding to remain an active participant in AI-generated codebases — "if you're lacking that fluency, your ability to participate in the project is meaningfully limited." The BVP Bessemer piece (June 10, based on Shopify and Ramp practices) independently converges: "Abdicate the toil. Never abdicate the thinking." Shopify practice: weekly demos (not velocity metrics) to test real comprehension. Named companies: Shopify parallel agents (10 simultaneous), Ramp Inspect agent (30% of merged PRs reviewed by AI). Shopify AI code reversion rates at baseline parity with pre-AI code — the first independent validation of production quality parity.
**Evidence level:** Level 2 (two independent named practitioner sources converging on cognitive debt failure mode; Shopify + Ramp named with metrics)
**Key claims:**
- Cognitive debt = understanding drifting from AI-built code = #1 AI-native team failure mode
- BVP + Willison arrive independently: understanding is prerequisite to direction, not optional review
- Shopify: AI reversion rate at baseline parity (first quality-parity validation)
- Ramp: 30% of merged PRs reviewed by Inspect agent
- Weekly demos (not sprint velocity) = comprehension test for AI-native teams
