## What "theory" means here (the frame)

- **Theory = the generating function behind the moves** — the compressed, named model that lets a practitioner act correctly in a situation the training never showed them. Three rents it must pay: **predicts · generates the next move · grounded** (pays rent in evidence, not vibes). The failure mode is *theory theatre* (SAFe: looks authoritative, unfalsifiable dogma).
- **The field is theory-*borrowing*, not theory-*less*.** No canonical theory of agentic engineering exists on the internet — but the explanatory theory already exists, filed under control theory, organisational learning, economics. **The white space is the *assembly*** — wiring borrowed + produced (our research's laws + groundwork's patterns) + evidence (the leg that adjudicates). *Don't claim the desert is empty; claim you drew the first map of it. The map is the moat.*
- **The empty-box rule governs delivery.** Understood-but-not-acted = nothing transferred. Theory reaches the student **derived** (they name the law after they've felt it) or as a **governor** (a pre-action question) — *not* front-loaded as a lecture.
- **One theory, two surfaces.** IC / CTO / field want incompatible *forms* of the same theory. Resolve: a named, scar-backed field-facing theory (groundwork = IP moat) whose classroom delivery is a derivation scaffold (AE101).
- **The same-hole insight.** Groundwork is a pattern language with lexicon + grammar but few *sentences* (it generates cards, not compositions/paths). AE101 lacks a theory layer. AE101's module *sequence* already IS a composition over groundwork's patterns. Wiring them fills both holes at once.

### 0 · Axioms — where the whole thing is written from `[grnd]`
- Capability is forged *only* by deliberate action under uncertainty → learning → capability → braver action. **Capability is a flow, not a stock.**
- Competence is the gate and can't be outsourced; practice before platform; the org's **learning rate is the ceiling**.
- A move only counts when it **crosses the wall** (personal → team → company); give it away so it compounds.

### 1 · Laws of the loop — how a single agent loop behaves and fails (the "physics")
- ★ The agent loop **is a closed-loop controller** (act→observe→correct). No feedback signal (test/eval/compiler) → it diverges. `[borrow:control theory]` `[curr:the-agent-loop]`
- The **Good Regulator**: to steer a system the controller must *contain a model of it* → why CLAUDE.md / context files work. `[borrow:Conant–Ashby]`
- **Requisite variety**: controller's repertoire must match task variety → narrow agents fail on varied work; fan-out splits variety into channels. `[borrow:Ashby]`
- Context is a **bandwidth-limited channel** — every irrelevant token is noise; recall rots as the window grows ("context rot"). `[borrow:Shannon]` `[curr:token-efficiency]`
- The **three failure modes** — goal drift / context rot / plausible-but-wrong — are *consequences of a statistical machine with a finite window*, not bad luck. `[curr]` — PROMOTED to anchor form 2026-07-02 → `curriculum/lectures/reading-the-return.md` (per-mode mechanism sentences + *Why these three, every run* anchor slide: law claim · Work-phase map anchor · enacted which-lens governor)
- ★ **Local success / global drift** is THE core agentic failure mode: agent wins the function, loses the system. `[curr:clean-code-is-steering]`

### 2 · Laws of autonomy — when/where to trust it unsupervised (the "engineering limits")
- ★ **Compound-reliability floor: 0.85ⁿ.** Chain *length*, not single-step quality, kills autonomy. `[rsch]`
- **Hybrid beats autonomous (today):** AI generates → human validates → AI executes → human monitors. `[rsch:L3]`
- ★ **Convergence Triad:** autonomy reaches production only where rules codified + correctness independently verifiable + talent constrained. A go/no-go diagnostic. `[rsch:L4]`
- ★ **Absorption bottleneck:** once generation is cheap, the binding constraint is the human's *evaluation rate*. 95%-at-volume worse than 80%-digestible. `[rsch:L4]`
- **Blast radius / error budget:** grant autonomy where downside is bounded and reversible; spend it against a measured failure rate. `[borrow:SRE]`
- ★ **Principal–agent:** "agent" is the literal economic relationship — delegated authority + hidden action + goal misalignment. Alignment, verification gates, reward-hacking are one frame. `[borrow:economics]`

### 3 · Laws of compounding — how capability is built over time (the "growth dynamics")
- **OODA with Act collapsed:** agents make acting nearly free → judgment (Orient) is the ceiling, not keystrokes. `[borrow:Boyd]` `[curr]`
- **Feedback latency dominates:** cycle *speed*, not generation speed, is the competitive variable. `[borrow:DORA]`
- ★ **The compound ladder:** fix → memory → skill → system. One cycle is a fix; a thousand is a system that learns faster than the field moves. `[curr]`
- ★ **Double-loop learning:** single-loop fixes the failing test; double-loop changes the *rule* that let the bug recur (= `/compound`, mechanised). `[borrow:Argyris]`
- **Variation–Selection–Retention:** generate variants → eval selects → memory retains. Throughput + a cheap selection filter beats first-try correctness. `[borrow:Campbell BVSR]`
- **Don't tamper:** change the process on a *recurrence* (special cause), not one stochastic miss (common cause). `[borrow:Deming/SPC]`
- ★ **Amplification:** AI amplifies the system it enters, doesn't transform it. Strong teams pull away, weak fail faster, median flat. `[rsch:L3]`
- **Individual ≠ institutional:** personal gains don't aggregate to org value without organisational change. `[rsch:L3, negative law]`
- **Competence-first sequence:** competence → discovery → context → compounding. Can't design an org for a capability nobody in it has yet. `[rsch:L3]`

### 4 · The crossing — personal → team → company (where transformations die)
- The **wall** is real: individual gains ≠ institutional value (productivity paradox). `[rsch]` `[grnd]`
- **Access–Trust gap:** experimentation always widespread, production-trust always rare, gap always >35pts. Bottleneck is trust, not access. `[rsch:L3]`
- **Trust is earned through observed competence**, not claimed capability — the crossing is demonstrated, not announced. `[rsch:L1, thin]`
- ★ **The value cycle:** Intent → Context → Work → Verification → Absorption → Outcome, compounding as it climbs personal→team→company. `[grnd]` — the natural spine.

### 5 · Vocabulary & typologies — the map and the names you can carry (the cohort's missing piece)
- **eval = judge / verifier / gate** — one primitive, three faces. `[curr]`
- **Three-block memory.** `[curr]`
- The **L0 → L3 ladder** — a typology of the engineer's own progression. `[curr]`
- The **pattern layer ontology:** Pattern / Component / Dish / Foundation. `[grnd]`
- **Steering is executable constraint; your stance is the ceiling** (the "infinite chameleon"). `[curr]`
- **Coding agents are the meta-platform:** they build the next agent; vendor platforms can't extend themselves. `[rsch:L3]`

### 6 · Governors — theory in operation (the pre-action questions that *are* the theory being used)
- **Name the uncertainty before you move;** aim at the unknown that will teach you most. `[grnd]`
- **Is this process rules-based, verifiable, talent-constrained?** (Triad as a go/no-go.)
- **What's the blast radius if it's wrong?** (sets the autonomy you grant.)
- **Comparative advantage:** give the agent the work where its *relative* cost is lowest — even where you're absolutely better. `[borrow:Ricardo]`

**Caveats.** A borrowed piece earns its keep only if the course can show **a scar it actually walked** — else it's analogy-theatre. Research-tagged laws carry evidence levels for a reason: L1 (trust-mechanism) is a candidate, not load-bearing. House-thesis pieces (competence-first, conditions-creator) carry confirmation-bias risk — validate adversarially.

## The solution — spine + body + doctrine

The theory doesn't need to grow; it needs a **spine** and a **body**. Two artifacts, one delivery doctrine — each aimed at one of the two failures.

**1 · Spine — the Field Map.** One durable artifact assembling the backbone laws onto groundwork's value cycle (the tight co-generation, doing real work). The coat rack: turns *"too much material"* into *"material with a place."* Given day one as orientation — *here's the whole territory; today we're here* — and every module references its position. Fixes **no-spine**.

**2 · Body — per-module law-anchors.** Each module gets one durable, projected, student-*owned* surface (the "slides" they asked for), shaped as map-anchored law cards: the ≤3 laws this module installs · their position on the map · the mechanism (WHY) · the governor (the pre-action question). The theory now survives imperfect delivery and is re-readable on Tuesday. Fixes **no-durable-body** (verbal-only).

**3 · Doctrine — map-first, derive-within.** Narrow, deliberate exception to *"tell nothing, name after"*: front-load the *map* (container — reduces load); still *do* the moves and name them against the map afterward (content — worldview-pure). Structure up front; empty box avoided.

### The Field Map (assembled spine)

```
  ┌──────────── THE LOOP CLIMBS: personal → team → company ────────────┐
  INTENT ─▶ CONTEXT ─▶ WORK ─▶ VERIFICATION ─▶ ABSORPTION ─▶ OUTCOME ─╮
     ▲                                                                │ compounds back
     ╰────────────────────────────────────────────────────────────────╯
  SOIL (axioms, spanning): action-under-uncertainty · competence-is-the-gate · cross-the-wall
```

| Phase | Laws that live here | The move it governs |
|---|---|---|
| **Intent** | name-the-uncertainty ★gov · comparative advantage · *(frontier: intent-scarcity)* | Aim the work at the unknown that teaches most; hand the agent what it's relatively best at. |
| **Context** | Good Regulator (CLAUDE.md = a model of the system) · context-as-bandwidth / context-rot · requisite variety | Give the controller a model; keep the channel clean. |
| **Work** | ★ agent-loop = closed-loop controller · ★ local-success/global-drift · three failure modes · steering-is-executable-constraint · OODA-with-Act-collapsed | Steer; watch for drift; your stance is the ceiling. |
| **Verification** | eval = judge/verifier/gate · jidoka/stop-the-line · ★ 0.85ⁿ reliability floor · hybrid>autonomous · ★ convergence-triad (trust gate) · blast-radius · ★ principal-agent | Build the selection filter; decide how much autonomy to grant. |
| **Absorption** | ★ absorption-bottleneck (the binding constraint) · ★ amplification · don't-tamper (SPC) | Match output to your evaluation rate; change the process on recurrence, not noise. |
| **Outcome / Crossing** | ★ compound-ladder · ★ double-loop learning · VSR · individual≠institutional · access-trust gap · learning-rate-ceiling | Encode the win so it compounds and crosses the wall. |

Two findings fall out of the assembly: **Verification is by far the densest phase** — a teachable claim in itself (agentic engineering is mostly a *verification* discipline; generation was never the hard part). And the dose governs itself: **one** map, **≤3** laws per module, theory as container not content.

**Third finding (2026-07-03, Antti + external read): the whole map is a feedback control loop.** Not only the inner agent loop (§1's closed-loop-controller law) — the *entire* Field Map is the canonical closed-loop negative-feedback diagram, and the near/far seam IS the feedforward/feedback boundary. Near half = feedforward (shape the run before it moves: intent, context, plan, memory). Far half = feedback (observe what happened, compare to target, correct, encode). This retro-explains a seam we drew on instinct — the strongest evidence the borrow fits: an outside theory predicts a split we already made. Verification stops being QA and becomes the **sensor layer** — a run with no verifier is open-loop (send energy, hope); a packaged run is closed-loop (reference · plan · verifier · judge · gate), so the system can feel error and respond.

Canonical mapping (design-side; jargon stays HERE, never reaches students):

| control-theory term | AE101 |
|---|---|
| reference / setpoint | spec · plan · done definition |
| controller | human + Claude + rules + skills |
| plant / process | agent run over the codebase |
| actuators | tools · edits · shell · PRs |
| output | the produced change |
| sensors / measurement | tests · diff · eval · review · CI |
| error signal | gap between desired and actual |
| feedback loop | act → observe → correct → encode |
| disturbance | drift · context rot · prompt injection · stale rules |
| integral memory | compounding — errors become durable rules |

Sharp field-facing statement: **agentic engineering is feedback control for work done by non-deterministic agents.** Where the borrow breaks: control theory says nothing about taste, meaning, org adoption, or human judgment — absorption is partly control bandwidth but also social uptake, so the frame grounds the spine without claiming the whole territory (label: intuition-pump-plus, earns keep by move-generation + retro-explaining the seam). ## Exercise format refactor (2026-07-04)

**Why (the same rubric, pointed at the vehicle).** The theory work fixed the *intrinsic* half of the overwhelm — thirty techniques compressed to six map-slots, freeing capacity. The exercises are where the *extraneous* half lives, and it was never touched. Antti's decomposition: *"a lot of novel concepts inside big prompts + unfamiliar codebase + pressure to proceed — the sum of overwhelm alongside implicit theory."* All three are **extraneous load**, and by Sweller they share one pool with the germane load (schema-building) we actually want — so high extraneous means the learning doesn't stick even with perfect theory. The exercises took a *double* hit: the three extraneous sources on top of concepts that were present but **unmapped** (implicit theory). The map fixes the implicit-theory leak; cutting the three sources handles the extraneous.

**Current-state finding (M3/M5 audit, representative samples).** There is **no slide style for exercises.** Lectures got the slides-only conversion; exercises did not. The unit is the **Phase** (`## Phase N: <verb>`), a multi-minute block bundling setup prose + one-or-more `{{prompt:}}` includes + ad-hoc `>` notes. And the variation is exactly what "no format" produces: chunk unit drifts (phases / no-phases / timeboxed `(N min)` workshop style from Agents-101); prompt-to-chunk ratio drifts (Phase 1 of `diagnose-and-resend` wraps two prompts); titles drift (`# Diagnose and *re-send*` vs `# Exercise: Map the access surface`); widgets drift (`HOX`/`Session`/`Watch-for`/timebox); closers drift (What this sets up / What happened / Approve). **The map is absent from every exercise** — the exact place a student is heads-down in a big prompt and an unfamiliar repo (peak overwhelm) never names which cycle-phase it is standing in.

**The good news: it is a reshape, not a rewrite.** The before/after material already exists as prose. Every prompt in `diagnose-and-resend` is already a sandwich — a setup paragraph, the `{{prompt}}`, a read-and-push-back paragraph. Slide-chunking formalizes what is already there.

**Target format — the exercise "slide" is a prompt-frame, not a claim-beat.** A lecture slide is a bolded assertion; an exercise slide is built around an *action* (the prompt). So the unit is: **before-beat** (`##` naming the move + its map slot + the load-shed line: what you are NOT on the hook to hold — the codebase, the other moves) → **the prompt** → **after-beat** (what to read in the return + the one push-back). Putting-*down* is the point: every rubric here is really about what the student is allowed to stop holding.

**Rubric extension (twin of the theory test).** Theory's test: *at the worst moment, do I know where I am.* Exercise's test: **at the start of this exercise, do I know which slot I'm in — and what I'm allowed to not hold?** Same eval family (B · cognitive-load / spine-anchoring), pointed at the vehicle.
