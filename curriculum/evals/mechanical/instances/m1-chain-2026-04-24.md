## Findings for the exercises (ordered by severity)

1. **Ex3 gitignore-by-default claim is latent-buggy.** *"Confirm `.claude/settings.local.json` is gitignored (it usually is by default; verify)."* — only true when the student's machine has a global gitignore entry for it (via `core.excludesFile`). On a fresh laptop with no global gitignore, the file gets tracked. The Ex3 prompt should make the verification active: grep the repo's `.gitignore`, add `.claude/settings.local.json` if absent. On this run it was ignored only because Antti's `~/.config/git/ignore` covers it.

2. **Ex2 diff push-back beat doesn't fire on a one-line fix.** `isSolid` inversion is a single-operator diff. *"Find me one line you'd have written differently"* has no surface. Either Ex2's planted bug gets chosen for a richer diff (3–5 lines minimum where taste choices appear), or the Debrief names the absence and moves the push-back move to M2 explicitly.

3. **`/context` fidelity in harness is thin.** The harness substitute lists files; it doesn't simulate token accounting. For M1 this is fine (the teaching point is *"there's an unread slice"*), but for any future module that leans on percentage-consumed (M4's walk, M5's `/compact`), the harness needs a proper context-use model.
