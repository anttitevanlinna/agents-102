# AE-42: Daily totals run too high on days with refunds

**Reporter:** ops-rotation@acme
**Status:** Open
**Component:** daily-totals / src/totals.js

## Description

The end-of-day totals report has been overstating revenue on days that include refund line items. Ops first flagged it last week — the warehouse extract shows -125.00 for a refund line, but the daily summary shows the day's total as if that refund weren't there.

Repro from the test harness:

```
sum([100, -125, 50])
// expected: 25
// actual: 150
```
