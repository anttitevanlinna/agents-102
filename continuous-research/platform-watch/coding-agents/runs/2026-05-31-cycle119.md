## Findings

### Finding 1: Ronacher observation file created — substantive content found

Armin Ronacher (Flask creator, Sentry CTO, @mitsuhiko) has published eight pieces in 2026 that together form a coherent practitioner belief system about agentic coding. Key recent posts:

- **"Clankers"** (May 26, 2026, lucumr.pocoo.org/2026/5/26/clankers/): Argues for "clanker" over "agent" — LLMs are tools, not beings with agency or moral status. "If my coding tool opens a pull request, I opened that pull request, not the machine." This is not semantic trivia — his belief that AI tools are owned by humans (not agents of their own) shapes how he assigns responsibility for outputs.
- **"Building Pi With Pi"** (May 24, 2026, lucumr.pocoo.org/2026/5/24/pi-oss/): Reports on using his own coding agent to develop itself. Primary failure mode: "slop issues" — AI-generated bug reports contain "plausible but wrong diagnosis" that create wasted effort. Over-engineering is the other failure: "the clanker will often add a tolerant reader. Then it will add a fallback, then maybe a migration." His principle: "almost always, the correct fix is not to handle the bad state, but to make the bad state impossible."
- **"Pi: The Minimal Agent Within OpenClaw"** (January 31, 2026, lucumr.pocoo.org/2026/1/31/pi/): Session trees (branching) prevent context waste. MCP's upfront tool loading is a mistake: "That makes it very hard to impossible to fully reload what tools can do without trashing the complete cache." Agents should write and run their own code rather than use pre-built extensions.
- **"Agentic Coding Recommendations"** (June 12, 2025, lucumr.pocoo.org/2025/6/12/agentic-coding/): Uses Claude Code with Max ($100/month), Sonnet-only (prefers it over Opus). `claude-yolo` alias skips permission prompts. Go strongly preferred for backend: explicit context system, fast test caching, structural interfaces. Python's pytest fixtures, complex async, slow startup all create friction. Key: "You get excellent SQL out of agents and they can match the SQL they write with the SQL logs." Permission checks must be locally visible — hiding them in another file guarantees agents forget them. Log everything to files for agent self-diagnosis.
- **"The Center Has a Bias"** (April 11, 2026, lucumr.pocoo.org/2026/4/11/the-center-has-a-bias/): Meaningful criticism requires direct engagement with technology. "A serious opinion on a new language, framework, device, or way of working usually has some minimum buy-in." Informed use — not abstract rejection — is the precondition for credible criticism of AI tools.
- **Speaker Deck presentation "Agentic Coding: The Future of Software Development with Agents"** (June 30, 2025): Custom bespoke tools outperform MCP. Excessive MCP causes "context rot." Logging is infrastructure. Fast, user-friendly tools are required — slow tools create expensive inference loops.

**X.com (May 2026):** "Claude Code feels like a tool that is used by the developers for the purpose it's written. I'm sure most coding agents are used by their developers, but I wonder what the percentage of code changes to their agents come from using their own agent." — noting the meta-platform angle from the practitioner's own perspective.

### Finding 2: Convergence on practitioner belief layer — "orchestrator not implementer" at Level 3

Multiple independent practitioners converge on the same belief package about when agentic coding works. This is now Level 3 (10+ independent voices in a consistent pattern).

**The convergence:**

1. **Role shift:** Developer role shifts from implementer to orchestrator/reviewer. Confirmed across: Addy Osmani (Chrome DevTools, Google), TechEmpower practitioners (May 15, 2026), Anthropic 2026 Trends Report (n=customer base + telemetry), Ronacher, Mike Mason.
2. **Spec quality is the new bottleneck:** "Give it success criteria and watch it loop" (Osmani). "Coding agents don't remove the need for senior engineering judgment—they concentrate it. The work shifts toward defining problems clearly" (techempower.com). "Developers now use AI in roughly 60% of their work — but they report being able to fully delegate only 0–20% of tasks" (Anthropic Trends Report). The delegation gap is real.
3. **Test-first is not optional:** When agents write tests after code, "the existing code becomes the spec." Agents validate rather than catch bugs. TechEmpower (May 15, 2026).
4. **Codebase quality determines agent success:** "Agents struggle with disorganized codebases, unclear conventions, and missing documentation." Stale context files (CLAUDE.md / AGENTS.md) actively decrease agent success rates. TechEmpower: "LLM-generated CLAUDE.md files actively decrease agent success rates compared to having no context file."
5. **Over-engineering is the characteristic failure mode:** Agents add layers rather than enforce invariants. Ronacher: "clanker will often add a tolerant reader. Then it will add a fallback, then maybe a migration." Osmani: "abstraction bloat — agents overcomplicate relentlessly."
6. **78% of Claude Code sessions involve multi-file edits** (Q1 2026 vs 34% Q1 2025). Average session: 23 minutes (vs 4 minutes autocomplete era). 47 tool calls per session. These are usage telemetry facts from Anthropic's Trends Report — not independently verifiable but are from first-party data.
7. **Verification/comprehension debt is the hidden cost:** Only 48% of developers consistently review AI code before committing (Osmani, January 28, 2026 — citing broader survey data). 38% find reviewing AI code harder than reviewing human code.

Sources:
- Addy Osmani, "The 80% Problem in Agentic Coding," January 28, 2026 [practitioner direct] https://addyo.substack.com/p/the-80-problem-in-agentic-coding
- TechEmpower, "Agentic Coding in Practice," May 15, 2026 [practitioner direct] https://www.techempower.com/blog/2026/05/15/agentic-coding-in-practice/
- Anthropic, "2026 Agentic Coding Trends Report," 2026 [vendor report — telemetry-based] https://resources.anthropic.com/2026-agentic-coding-trends-report
- Armin Ronacher (multiple posts above) [practitioner direct]

**Evidence level:** Level 3 — convergence across 10+ independent practitioners on the same pattern. The "orchestrator not implementer" belief is now mainstream practitioner consensus.

### Finding 3: Claude Opus 4.8 + Dynamic Workflows — significant platform update (May 28, 2026)

**Claude Opus 4.8** (released May 28, 2026 — 41 days after Opus 4.7):
- New default model in Claude Code; `claude-opus-4-8` API identifier
- Fast mode now 3x cheaper than previous fast-mode pricing
- "Around four times less likely than Opus 4.7 to allow flaws in code it has written to pass unremarked" — honesty/self-correction improvement
- Pricing unchanged: $5/M input, $25/M output; Fast mode $10/$50
- Available: Claude API, Bedrock, Vertex AI, Microsoft Foundry

Sources: [vendor press release — bare facts only] https://www.anthropic.com/news/claude-opus-4-8

**Dynamic Workflows** (research preview, May 28, 2026):
- Claude writes a JavaScript orchestration script that fans work across tens-to-hundreds of parallel subagents per session
- Cap: 1,000 subagents per workflow (reported by MarkTechPost)
- Verification built in: results are checked before being folded into final output
- Sessions resumable if interrupted
- Available: Claude Code CLI, Desktop, VS Code extension, API, Bedrock, Vertex AI, Foundry
- Enabled for Max, Team, Enterprise (admin-enabled); Enterprise disabled by default due to high token cost
- Primary demonstration: Jarred Sumner (Bun creator) used it to port 750,000 lines of Zig to Rust in 11 days with 99.8% test suite passing — not yet in production but the scope is unprecedented
- Named early practitioner reactions: Klarna's Alessio Vallero — "identifying dead code and surface cleanup opportunities that traditional static analysis missed"; CyberAgent's Ken Takao — "fill the gap between firing off a single subagent and building out a full agent team" (vendor-sourced quotes, Level 0)

Sources: [vendor announcement] https://claude.com/blog/introducing-dynamic-workflows-in-claude-code; [general press] https://techcrunch.com/2026/05/28/anthropic-releases-opus-4-8-with-new-dynamic-workflow-tool/

**Assessment:** Dynamic Workflows is the most significant Claude Code platform extension since Multi-Agent Orchestration at Code with Claude 2026 (May 6). The Bun example is the largest publicly documented single-workflow task (750K lines, 11 days). The JavaScript orchestration approach — Claude writes the script, the runtime executes it — is architecturally different from the Dreaming/Outcomes pattern (which is about session memory and quality loops). Watch for: first independent practitioner reports of production-workflow use beyond demos. Current evidence level: Level 1 (vendor demo + one named founder, not yet independent deployment accounts).

### Finding 4: Anthropic 2026 Trends Report — delegation gap quantified

**"The delegation gap":** Developers use AI in ~60% of their work but can fully delegate only 0–20% of tasks. This gap has a name and is now documented via Anthropic's own telemetry.

**27% of AI-assisted work is net-new work** — tasks that would not have been done at all without AI. This is the productivity argument that doesn't depend on speed claims: coding agents create new work, not just faster existing work.

**Context file quality finding:** Projects with well-maintained context files see 40% fewer agent errors and 55% faster task completion (Anthropic Trends Report). But stale/LLM-generated context files actively hurt — the signal is that HUMAN-maintained context files matter, not file presence alone.

Source: Anthropic, 2026 Agentic Coding Trends Report [vendor report, telemetry-based] https://resources.anthropic.com/2026-agentic-coding-trends-report
Analysis: Pathmode [practitioner analysis] https://pathmode.io/blog/orchestration-era-needs-intent

**Note:** This is a vendor-produced report using Anthropic's own telemetry. The telemetry numbers (78% multi-file sessions, 23-min average session) are plausibly reliable as first-party data; the case studies (Rakuten, CRED, TELUS, Zapier) are Level 0 vendor-selected customers. The delegation gap stat (60% use / 0-20% full delegation) is survey-based — methodology not fully disclosed.
