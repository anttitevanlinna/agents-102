# sources/

Raw material the brain is built from. Starts empty on purpose — you fill it on training day, curated around the one challenge you named in prework.

**Three kinds of sources go here:**

- **Confluence / wiki pages** — exported as markdown or PDF. The pages from your company's knowledge base that actually speak to this challenge. Three to eight pages is plenty; more is noise.
- **Office365 material** — docs, decks, and emails from OneDrive / SharePoint / Outlook, saved out as PDF or markdown. Meeting notes, strategy docs, customer emails — whatever your colleagues have already written that's relevant.
- **Internet best-practice articles** — two to four pieces from practitioners writing about this exact problem. Not vendor blogs. Not analyst predictions. People who've actually done the work. Save as markdown or PDF.

**Keep sources flat.** Don't create subfolders — the brain's citation convention points to `sources/<filename>` and subfolders make those pointers brittle.

**Don't hand-edit source files during the training.** The whole value of the brain is that it's derived from sources you can always re-check. If something's wrong on a brain page, fix the brain page — or add a new source.

**Rename files to something readable** when you drop them in. `confluence-pricing-rfc-2026-q1.md` beats `export-47293.pdf`. You'll cite these by filename on every brain page, so pick names you can stand looking at.
