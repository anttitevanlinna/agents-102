### 4. Production Failure Patterns Are Universal Across Frameworks
**Source:** https://dev.to/synsun/autogen-vs-langgraph-vs-crewai-which-agent-framework-actually-holds-up-in-2026-3fl8 — [practitioner direct]

**Date:** 2026
**Agent level:** N/A (meta-finding)
**What:** Consistent failure patterns across all frameworks: (1) Tool calling fails 3-15% of the time. (2) Non-deterministic routing is the #1 debugging nightmare. (5) Cost overruns from uncapped revision cycles ($4 in tokens for a single pipeline run).
**Evidence level:** Level 2-3 (approaching convergence — multiple practitioners report identical patterns)

## Orientation

**The framework wars are a distraction.** The real findings are structural:

1. **Context engineering matters more than framework choice** (Level 3 convergence). This should reshape our advisory — CTOs asking "which framework?" are asking the wrong question.

2. **The business user gap is enormous.** Only 4 of 35+ platforms are accessible to non-developers. This is why vertical SaaS wins for business process agents — they fill the gap that open source can't.

3. **Dify (134K stars) and n8n (150K+ stars) are the actual volume players.** The developer framework debates miss the platforms where most agent-adjacent work actually happens. These deserve dedicated tracking.

4. **The stack is layering, not consolidating.** Model → Runtime → Framework → Agent. MCP + A2A protocols may make framework choice less important over time. The CTO should bet on the protocol layer, not any single framework.
