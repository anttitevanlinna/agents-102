## Mood landing per module (1-10 + evidence)

**M1 — joyful creation. Target: "I made this. It's me." Score: 8/10.**
Connections opens with the sponsor speaking first, then pivots to personal stakes: *"Was there ever a moment where it actually felt like yours, and if so, what made the difference?"* Exercise closes with "this is me" Phase 6. Bridge sells forward: *"You just made great output. But it's a one-shot."* Mood lands through the artifact (the site), not through a narration of mood. Losing one point because the sponsor-speaks-first opener front-loads gravity before the joy has had a chance to spark — M1 is meant to be the one session where the air is light. The sponsor sentence is the right move; its placement could let a lecture frame open first.

**M2 — satisfied compounding. Target: "it runs while I sleep." Score: 8/10.**
Connections nails the personal→work pivot openly: *"Module 1 was for you… Now we turn to work."* Good. Debrief extracts compounding as the thing that happened. The "it runs while I sleep" feeling is deferred to homework (scheduled agent), not the in-room beat — the in-room beat lands on *"persistence + automation = system"* which is cognitive satisfaction, not ownership-feeling. The mood contract wants the student walking out *feeling* it compounds. Currently they walk out having *built the scaffold that will compound overnight*. Close, but 2/10 of the contract is in the homework rather than in the body.

**M3 — unsettled competence. Target: "I wonder if this is right?" Score: 9/10.**
Best-landed mood in the arc. Debrief Q1: *"Where does the synthesized answer feel almost right but not quite? Name the shape of the doubt, not a specific fix."* Artifact = `module-3/wonder.md` — a held question written to disk, not resolved. Bridge escalates rather than soothes: *"it just gained access to three systems. What's the worst thing it could do with that?"* This is the reference pattern for the arc. Other modules should envy this.

**M4 — deepened unease. Target: "Damn, this is complex stuff." Score: 8/10.**
Mood contract is explicit about the distinction between *"I'd like to be safe and I genuinely don't know how to know"* (epistemic) vs. *"I screwed up"* (punitive). Module file follows it: *"'I can't tell' is a real answer"*, residual-risk-as-artifact, Debrief Q1 surfaces the uncomfortable row. Teaching moment names it as a room-level pattern: *"The variance across the room IS the teaching moment."* Works. Dock one point for the word "plain/plainly" appearing 3× in the stretch between M3 and M4 ("plain note of what feels off," "plain state," "leaving it plain is the discipline") — not wrong in any one spot, but starts reading like a tic.

**M5 — mechanical rescue. Target: "Ahh, this is actually fixable." Score: 9/10.**
Rescue is earned and bounded — two things that usually don't co-exist. Maintainer section is explicit: *"Do NOT resolve M3's strategic uncertainty or M4's security residual. Only groundedness-for-this-shape-of-output gets rescued."* Student-facing text delivers the scoreboard moment (*"the scoreboard IS the explanation"*) and the bounded-rescue move (*"What specifically won't this judge catch?"*). The "Known limit:" line is mandatory in the saved judge. Arc hygiene is excellent here.

**M6 — unleashed leverage. Target: "we can automate the loop." Score: 9/10.**
Connections captures the mood pivot in the cleanest prose of any module: *"What if the judge kept getting better — not because you edited it, but because it watched itself miss things? What if you weren't in the room while that happened?"* Walk-away window = first true trust-distance moment. The Key Concepts ("eval as infrastructure," "two loops compounding," "walk-away autonomy") carry the leverage feeling. Bridge flips forward into M7: *"Who else on your team would want this?"* — seamless mood handoff.

**M7 — generous impulse. Target: "this is starting to work. I wonder if others could benefit?" Score: 8/10.**
Connections does the work: *"The question that shows up on its own: could this help someone else?… Hold that feeling. It's not a governance question yet. It's a generous one."* Exactly the contract. JTBD reframe ("interview for the job") is the 2026-04-20 move and lands — positions the teammate's job before the student's solution. Minus points: the three-walls pre-read is front-loaded with Access-Trust Gap data and "54-95% access, 5-22% production trust," which is a research-y frame that can cool the generous impulse into a compliance posture before the exercise starts. The contract warns: *"a system shared from enthusiasm promotes differently from one shared from obligation."* The statistic risks flipping it.

**M8 — awe and curiosity. Target: "oh shit — where is this all going?" Score: 6/10.**
**This is the arc's weakest landing.** The module file is structurally sparse — no `## Connections` section, no `## Debrief` section, only a Bridge. Compare against every other module's four Cs spine. The content-strategy contract specifies the Connections opener (*"You've each built agents for one part of the initiative. What happens when they all run together?"*) but the module file omits it. The Identity-naming close is well-written — *"You are now agent builders"* — but it lands *after* a Live Deliberation section that is explicitly tentative and flags four open questions including runtime dependency. The student-facing body reads, in places, like an internal design memo ("Not committed. Flagged here so the direction is visible..."). Awe is hard to generate from a module that is openly uncertain about its own peak. The flywheel needs a delivery, not a disclaimer.

## Seam transitions (7 seams)

**M1 → M2: Clean.** Explicitly named in M2 Connections (*"Module 1 was for you… Now we turn to work"*). The prework/Module-1/Module-2 folder seam is coherent. The bridge sentence *"What if it could remember, grow, and compound?"* is answered by M2's first move. No premature resolution.

**M2 → M3: Clean.** M2 ends with the crux skill previewed lightly; M3 opens with *"Last module you were the librarian… Today you hire three agents to do that search."* The "job is getting too big for one agent" bridge is the right motivation. No leakage.

**M3 → M4: Clean.** M3 bridge: *"it just gained access to three systems. What's the worst thing it could do with that?"* M4 Connections picks up: *"You've had three agents search your company and three more decide… Name the thing you'd want to check first — from a risk angle."* The M3 unease stays unresolved going into M4 (correct — M5 owns the rescue), and M4 adds its own distinct flavor of unease (epistemic). This is the cleanest cross-mood seam.

**M4 → M5: Clean. Slight jargon leak.** M4 bridge: *"the residual risk is named. But the output inside the scope — can you trust what it actually says?"* M5 Connections picks up the M3 feeling directly: *"the synthesized answer sat at an uneasy distance."* Good. Minor: "residual" from M4 is assumed familiar in M5 debrief discussion (not a blocker — the word is common). M5 mood contract is explicit about not rescuing M3 or M4's residual — and the module honors it.

**M5 → M6: Clean.** M5 bridge: *"five claims, four detectors, one judge. Now imagine the gold standard has three hundred claims… That's evals."* M6 Connections lands exactly there: *"Today the question shifts. What if the judge kept getting better…"* The M5 artifact (`judges/groundedness-judge.md`) is state-carried — M6 maintainer section flags it as *"hard dependency."* This is state-compound done right.

**M6 → M7: Mostly clean, minor vocabulary jump.** M6 bridge: *"Who else on your team would want this?"* M7 Connections builds: *"Your memory has been working for you… could this help someone else?"* Mood handoff works. But M7 introduces three heavy terms in close succession in the lecture frame — *absorption bottleneck*, *Access-Trust Gap*, *discoverability* — none of which have appeared in M1-M6. Three research-derived terms hitting the student in the first ten minutes of a module whose mood contract is generosity is a tonal mismatch. See Fix #3.

**M7 → M8: Broken.** This is the most severe seam problem. M7 bridge: *"What if you didn't have to build each new agent by hand?"* — well-aimed. But M8 module file has no Connections section at all, so the pickup fails structurally. The Tentative Live Deliberation section then reads like the module apologizing for itself before delivering its peak. A student arriving at M8 after a strong M7 close lands in an unanswered room. See Fix #1.

## State carry-over

**M1 → M2 artifact compound: Correct.** M1 produces `module-1/CLAUDE.md` (scoped). M2 introduces the root `CLAUDE.md` via the starter zip, *separate from* Module 1's file (no clobber). The substrate continuity (guardrails as discipline) compounds; the subject pivots (personal → work). Explicitly named in M2 Connections. Clean.

**M2 → M3 memory compound: Correct.** M3 runs on M2's memory + sources + challenge. No reset. Three retrievers point at the same three source zones curated in M2. CUSTOMER note in content-strategy: *"No additional data pre-load. Connectors already in place from Module 2."* Clean.

**M3 → M4: Correct.** M4 audits the M3 system. *"Run two pre-made skills against your Module 3 system."* No reset.

**M3/M4 → M5 residual doubt compound: Correct and explicit.** M5 Connections directly names: *"In Module 3 the synthesized answer sat at an uneasy distance."* The M3 `wonder.md` artifact and M4 `residual.md` artifact persist; neither is resolved in M5. M5's rescue is explicitly bounded to groundedness-of-this-output. This is the highest-stakes carry-over in the arc and it is handled well.

**M5 → M6: Hard file dependency.** `judges/groundedness-judge.md` produced in M5 is consumed as infrastructure in M6's exercise. Maintainer-side callout owns the dependency. Clean.

**M6 → M7: State carry is soft.** M7 references the "memory has been working for you for a day and a half" but does not consume a specific M6 artifact (judge, eval loop). This is fine — M7 is a strategy module, not a technical continuation — but the state is a felt state, not a file state. The contract warrants this shift.

**M7 → M8: Fragile.** M8 exercise description says *"Everyone's agents run on shared context"* but the module file doesn't specify whose context, in which folder, how the M1-M7 accumulation ("memory, sources, skills, evals, sharing protocol") is loaded. A student reading the committed M8 exercise alone can't tell whether their accumulated stack is central or decorative. See Fix #2.

**M1-M7 → M8 accumulated stack:** Named in content-strategy as the point of M8. Partially present in the module file (*"the company's own strategic context feeds into all agents"*). Not structurally enforced in the exercise instructions.

## Severity-ranked fix list (top 5)

**1. [Severity 5, arc-breaking] M8 module file is structurally incomplete.**
Module(s): M8. No Connections section; no Debrief section; Tentative Live Deliberation section reads like internal design notes in a student-facing file. The arc's awe-peak module has the sparsest spine in the training.
Fix: Add `## Connections` (content-strategy has the opener verbatim), `## Debrief` (pattern from M6/M7), and move the Tentative Live Deliberation block into a maintainer-space comment or a sibling design-memo file. Student-facing M8 should commit to one peak (current Joint Double Diamond or live deliberation) and ship it without self-doubt in the text. Must fix before manual run.

**2. [Severity 4] M8 does not structurally consume the M1-M7 accumulated stack.**
Module(s): M8. The exercise says "everyone's agents run on shared context" but doesn't name the files, folders, or artifacts that arrive. A student reading M8 alone cannot tell which of their accumulated artifacts (memory, sources, judges, skills, sharing patterns) are load-bearing.
Fix: Add a "What your agent arrives with" preamble to the Joint Double Diamond exercise, naming the specific paths from M2 (`memory/`, `sources/`), M4 (`skills/`), M5 (`judges/groundedness-judge.md`), M6 (eval infrastructure), M7 (sharing plan) and what each is used for in Diamond 1/2. Without this, state carry-over is implicit and the flywheel claim is asserted rather than shown.

**3. [Severity 3] M7 pre-read front-loads research statistics that cool the generous impulse.**
Module(s): M7. The Access-Trust Gap numbers (54-95% access, 5-22% trust, 35-point gap) arrive in pre-read and lecture-frame before the exercise has earned the mood. The contract says *"shared from enthusiasm, not obligation."* The statistic is an obligation frame.
Fix: Move Access-Trust Gap from pre-read to the Debrief or Concept section — *after* the student has done the JTBD interview and felt the gap for themselves. Let the feeling come first, the statistic second, per the M4 pattern ("variance across the room IS the teaching moment"). Don't open M7 with compliance math.

**4. [Severity 2] "Plain/plainly" tic across M3/M4/M7.**
Module(s): M3, M4, M7. Six uses in student-facing copy. Memory flags this as an LLM tell word.
Fix: One-pass search-and-replace. Substitutes: "straight," "truthful," "real," or cut entirely. Most uses are intensifiers that the surrounding sentence doesn't need.

**5. [Severity 2] M2 mood lands in the head, not the body.**
Module(s): M2. The "it runs while I sleep" feeling is delivered as homework (schedule your agent). In-room, the student walks out with a built system but not yet a *compounding* system.
Fix: Either (a) have the scheduled-agent setup happen in-room as a 10-minute add to the memory exercise so the student leaves having seen the first run, or (b) accept the homework-deferral and adjust the mood contract to "ownership of a system that will compound overnight" rather than "felt compounding in-room." The cleaner move is (a) — the contract is right; the delivery shifts.

## Verdict

**APPROVE WITH TODOs.**

The arc is coherent, the mood contracts are real and mostly honored, the state carry-over is legible through M7, philosophy is earned-not-announced throughout, and the central rescue seam (M3/M4 → M5) is one of the best-engineered mood handoffs I've read in a curriculum document. M3, M5, and M6 would each stand as a reference for how to write a deliberate-mood module.

The arc-breaking fix is M8 (Fixes #1 and #2). The module file is currently incomplete against its own content-strategy contract, and the tentative Live Deliberation text reads like the authors hedging in the student's peripheral vision. The awe-peak module can't afford visible uncertainty about its own peak — that steals the mood before the exercise starts. Either commit to the current Joint Double Diamond or ship the live-deliberation variant; don't publish both in the same student file.

Fix #3 (M7 statistic placement) is the next-most-impactful because M7 is the inflection into generosity, and the tonal miscue there rebounds into M8's awe-landing. Fixes #4 and #5 are polish.

**Minimum set to reach full APPROVE:** Fixes #1, #2, #3. #4 and #5 can ship with the first delivery and get cleaned in the first retro pass.

Proceed to manual run after Fix #1 + #2 land. Fix #3 is worth doing before the manual run but is not a blocker — a facilitator can compensate live in a way Teacher Claude can't compensate for a missing M8 Connections.
