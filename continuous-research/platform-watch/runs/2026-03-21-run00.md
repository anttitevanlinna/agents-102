## Findings

### Codex has evolved into a cloud-parallel coding agent with mid-task steering
**Source:** https://openai.com/index/introducing-gpt-5-3-codex/ — [vendor press release]
**Date:** February 2026
**What:** GPT-5.3-Codex (Feb 2026) is the latest model, combining coding + reasoning in one model, 25% faster, fewer tokens. The key new capability is interactive mid-task steering — you can redirect the agent while it works without losing context. Codex app (March 2026) adds worktrees (multiple agents on same repo), skills (extending beyond code), and automations (unprompted routine work like CI triage).
**Evidence level:** Level 0 (vendor announcement — capabilities claimed, not independently verified)

### Practitioner reports 85-90% success rate on well-scoped maintenance tasks
**Source:** https://zackproser.com/blog/openai-codex-review-2026 — [practitioner direct]
**Date:** ~Early 2026
**What:** Zack Proser (WorkOS engineer) reports Codex success rate improved from 40-60% (May 2025) to 85-90% for well-scoped maintenance work. Uses a two-tier workflow: queues 3-5 Codex tasks for routine work (TypeScript fixes, dependency updates, test coverage), then switches to Claude Code or Cursor for harder architectural problems. Calls it "production-ready infrastructure that fundamentally changed how I build software."
**Evidence level:** Level 2 (single practitioner experiment, specific and detailed)

### Codex vs Claude Code: convergence in capability, divergence in philosophy
**Source:** https://www.builder.io/blog/codex-vs-claude-code — [practitioner analysis]
**Date:** ~Early 2026
**What:** Builder.io's comparison finds the tools converging in capability. Key difference: Codex offers granular reasoning controls (low/medium/high) and the same company controls both tool and model. Claude Code is more autonomous/"hand off the task" oriented. Builder.io user sentiment data rated GPT-5 40% higher than GPT-5 Mini and Claude Sonnet. Neither tool holds a decisive lasting advantage — feature parity is the real story.
**Evidence level:** Level 2 (single company's analysis with user data)

### Operator deprecated, replaced by ChatGPT Agent (browser automation unified into ChatGPT)
**Source:** https://openai.com/index/introducing-chatgpt-agent/ — [vendor press release]
**Date:** July 2025
**What:** Operator (launched Jan 2025, CUA-powered browser agent) was deprecated Aug 2025 and folded into "ChatGPT Agent" — a unified system combining Operator's browser interaction, deep research, and conversational AI. OpenAI also launched Atlas (Oct 2025), a dedicated agentic browser. The browser-agent space is now crowded (Google Project Mariner, Amazon Nova Act, Microsoft Computer Use for Copilot Studio).
**Evidence level:** Level 0 (vendor announcement)

### Agents SDK and Responses API are the developer-facing agent infrastructure
**Source:** https://github.com/openai/openai-agents-python — [practitioner direct / open source]
**Date:** Ongoing (launched March 2025)
**What:** OpenAI's agent building stack: Responses API (agentic loop with built-in tools — web search, file search, computer use, code interpreter, MCP) replaces both Chat Completions and Assistants API (deprecated Aug 2025, sunset Aug 2026). Agents SDK is open-source, provider-agnostic, supports handoffs between agents, guardrails, tracing. AgentKit adds visual Agent Builder canvas and Connector Registry for enterprise.
**Evidence level:** Level 0 for AgentKit announcement; the SDK itself is open-source and inspectable

### Key limitations: no model selection, usage caps, fails on architectural complexity
**Source:** https://zackproser.com/blog/openai-codex-review-2026 — [practitioner direct]
**Date:** ~Early 2026
**What:** Practitioners report: (1) no control over which model handles your task — Codex picks internally, (2) usage limits frustrate heavy users — some report hitting code review caps quickly, (3) still struggles with architectural decisions and security-critical code where every line must be inspected, (4) GPT-5.3-Codex API access not yet available. GPT-5.3-Codex is first model OpenAI classifies as "High" cybersecurity risk.
**Evidence level:** Level 2 (practitioner experience, corroborated by community forum reports)

### SWE-Bench Pro: 56.8% — impressive but still fails ~43% of professional tasks
**Source:** https://openai.com/index/introducing-gpt-5-3-codex/ — [vendor press release]
**Date:** February 2026
**What:** GPT-5.3-Codex sets industry high on SWE-Bench Pro at 56.8% and leads Terminal-Bench 2.0 at 77.3% (vs Claude's 65.4% on Terminal-Bench). While industry-leading, the ~43% failure rate on SWE-Bench Pro means it is a powerful collaborator, not a replacement. These are vendor-reported numbers.
**Evidence level:** Level 0 (vendor-reported benchmarks — independent verification needed)

## What I Looked For But Did Not Find
- **Independent benchmark verification** of GPT-5.3-Codex claims (SWE-Bench Pro, Terminal-Bench). All benchmark numbers trace back to OpenAI's own announcements.
- **Enterprise deployment case studies** — no non-vendor evidence of organizations deploying Codex at scale for production engineering workflows. Practitioner evidence is individual developers, not teams.
- **Pricing granularity for enterprise/API** — specific API pricing for GPT-5.3-Codex not found (API access reportedly not yet available).
- **Non-coding agent experiences** — Codex is purely a coding agent. OpenAI's broader agent ecosystem (ChatGPT Agent, Atlas browser) had no practitioner-level reviews found.
- **Nordic-specific adoption data** — no evidence of Nordic companies using Codex in any distinctive way.
- **Counter-evidence or negative practitioner experiences** beyond the limitations already noted. Most criticism is "it's good but not autonomous" rather than "it failed badly."

## Orientation

OpenAI's agent ecosystem is bifurcating: Codex for coding (cloud-parallel, increasingly capable at routine tasks, still weak on architecture), and ChatGPT Agent/Atlas for browser-based tasks (unified from Operator). The developer infrastructure (Responses API + Agents SDK) is maturing and deliberately open/provider-agnostic. The key gap for our platform advisory work: all strong evidence is from individual practitioners, not enterprise teams — the chasm-crossing signal (first teams in proper firms reporting success) has not yet appeared in public for Codex specifically.
