## Findings

### Finding 1: Claude Code Engineering Team Org Restructuring (AI-Native Teams Level 2)
**Source:** https://claude.com/blog/running-an-ai-native-engineering-org — [practitioner direct, Anthropic Engineering]
**Date:** June 4, 2026
**Agent level:** team
**What:** Fiona Fung (Director of Engineering, Claude Code + Claude Cowork) documents exactly how the Claude Code engineering team restructured when agentic coding became the default working mode. Four process areas rewrote themselves: planning, context gathering, code review, and team composition. The bottleneck shifted from "writing code" to "everything around code" — verification, review, security, cross-functional coordination.
**Evidence level:** Level 2 (single team, internal Anthropic — strong practitioner-direct account; caveat: Anthropic is the platform vendor, internal team incentive to publish positive account)
**Key claims:**
- "Writing code, writing tests, and refactoring rarely slows us down anymore" — the OLD bottleneck is gone
- **Planning:** Moved from 6-month roadmaps to just-in-time (JIT) planning. Original roadmap "was out of date by month three" due to increased engineering velocity. Shifted to prototyping + rapid internal feedback loops.
- **Context gathering:** Default shifted from "ask the code author" to "ask Claude first, then determine whether the inquiry could be automated entirely." Human SMEs now consulted for edge cases, not routine context.
- **Code review:** Human review shifted from universal oversight to targeted expertise. "Claude handles all the style and linting... Where we still definitely want a human is expertise." Domain specialists (legal, security, product-sense) are now the review layer — not general code review.
- **Team composition:** Roles blurred significantly. PMs now prototype code; engineers handle design work. Hiring shifted toward "creative builders with product sense" and "engineers with deep systems expertise" — not raw throughput.
- **Zero non-Claude commits in four months:** Quantified threshold — the team hit a point where non-AI-assisted commits are the exception, not the norm.
- **Onboarding acceleration:** Engineers ship real code within week one.

### Finding 4: AI as Critical Infrastructure — Claude Outage June 2 Lens
**Source:** https://www.thoughtworks.com/en-us/insights/blog/generative-ai/claude-outage-june-2026 — [domain trade publication, Thoughtworks]
**Date:** June 2026
**Agent level:** company
**What:** Thoughtworks published analysis of the June 2, 2026 Claude outage (elevated error rates across Opus 4.6, Claude API, Claude Code CLI). Development velocity dropped, customer support systems failed, data pipelines froze. Ken Mugrase (Thoughtworks) argument: teams are treating AI as critical infrastructure without the resilience planning that implies. Three recommendations: graceful degradation, dependency auditing, AI-specific observability.
**Evidence level:** Level 2 (single outage analysis, practitioner-level; Thoughtworks independent of Anthropic)
**Key claims:**
- June 2 outage affected "everything from Opus 4.6 to the Claude API and the Claude Code CLI"
- Business impact: dev velocity drop, customer support failures, data pipeline freezes
- Mugrase: "AI tools should amplify engineers' capabilities. It shouldn't act as a structural crutch." — the infrastructure dependency was unexpected in speed of accumulation
- Frames the evolution: coding tool → engineering dependency → critical infrastructure (with the resilience planning gap to match)
