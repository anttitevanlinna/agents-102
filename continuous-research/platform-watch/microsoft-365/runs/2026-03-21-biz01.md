## Findings

### 1. Copilot Studio Lite vs Full: the business user ceiling is low
**Source:** https://www.hso.com/blog/microsoft-copilot-vs-studio — [practitioner analysis / consultancy guide]
**Date:** 2026
**Agent level:** Personal → Team (ceiling)
**What:** Business users can create "lite agents" inside M365 Copilot using natural language — grounded in SharePoint/M365 data, suitable for departmental Q&A bots. But they cannot independently integrate external systems (Dynamics 365, SAP, Salesforce), build multi-step approval processes, access detailed usage analytics, or manage environment isolation. When agents need data outside M365, approval workflows, or multi-step automation, business users hit a wall and need Copilot Studio full — which requires developer or power-user skills. The lite-to-Studio upgrade is one-click technically, but the skill gap is not one-click.
**Evidence level:** Level 2 (single consultancy analysis, consistent with Microsoft's own docs)

### 2. No real business process deployments found — only hypothetical examples
**Source:** https://www.orchestry.com/insight/top-copilot-agent-use-cases — [consultancy blog]
**Date:** 2026
**Agent level:** N/A

This is the most-referenced use case article and it contains zero evidence of production use.

### 3. Copilot Cowork: the personal agent play, built with Anthropic
**Source:** https://www.microsoft.com/en-us/microsoft-365/blog/2026/03/09/copilot-cowork-a-new-way-of-getting-work-done/ — [vendor blog, Level 0]
**Source:** https://fortune.com/2026/03/09/microsoft-copilot-cowork-ai-agents-anthropic-e7-m365-saas/ — [general press]
**Date:** 2026-03-09
**Agent level:** Personal
**What:** Copilot Cowork uses Anthropic's Claude to handle multi-step tasks across M365 apps — e.g., "prepare for a customer meeting by assembling a presentation, pulling financials, emailing the team, scheduling prep time." It creates a plan, runs in the background, checks in at clear checkpoints before applying changes. Currently in Research Preview with limited customers; broader Frontier program access late March 2026. This is the closest Microsoft has to a real personal agent (not chatbot) but it is not yet shipping broadly and no independent user reports exist.
**Evidence level:** Level 0 (vendor announcement + general press coverage, no practitioner validation)

### 4. E7 "Agent-First" tier: $99/user/month, launches May 2026
**Source:** https://fortune.com/2026/03/09/microsoft-copilot-cowork-ai-agents-anthropic-e7-m365-saas/ — [general press]
**Date:** 2026-03-09
**Agent level:** Company
**What:** Microsoft 365 E7 "Frontier Worker Suite" bundles E5 ($60) + Copilot ($30) + Agent 365 ($15) = $99/user/month vs $117 separately. Agent 365 is a new control plane for IT/security to observe, secure, and govern agents. Launches May 1, 2026. This is the first SKU that explicitly packages agent governance. The per-seat model means costs scale with headcount not usage — a concern for orgs where agent value is unevenly distributed across employees.
**Evidence level:** Level 1 (confirmed pricing from Fortune reporting on Microsoft announcement; no practitioner input on value)

### 5. Copilot reliability problems reported by business users
**Source:** https://learn.microsoft.com/en-us/answers/questions/5549599/persistent-crashing-issue-with-microsoft-365-copil — [practitioner direct, Q&A forum]
**Source:** https://concentric.ai/too-much-access-microsoft-copilot-data-risks-explained/ — [security analysis]
**Date:** 2025-2026
**Agent level:** Personal
**What:** Business users report Copilot crashes after 15-20 interactions (sometimes as few as 7), with complete memory loss after crash. Document modification tasks (editing Excel, PowerPoint, Word) described as "practically useless." Security researchers note Copilot inherits existing M365 over-permissions, making data leakage the primary risk — not a new vulnerability but an amplification of existing permission debt. Nadella himself reportedly described some integrations as "almost unusable."
**Evidence level:** Level 2-3 (multiple independent users reporting same reliability issues; convergence on "unreliable for real work")

### 6. Governance exists but personal→team promotion is not a designed path
**Source:** https://learn.microsoft.com/en-us/microsoft-copilot-studio/admin-share-bots — [vendor docs]
**Source:** https://www.microsoft.com/en-us/microsoft-copilot/blog/copilot-studio/6-core-capabilities-to-scale-agent-adoption-in-2026/ — [vendor blog, Level 0]
**Date:** 2026
**Agent level:** Team → Company
**What:** Copilot Studio has sharing controls (Editor/Viewer permissions, Managed Environments), and agents can be published to Teams channels. Microsoft Agent 365 (May 2026) promises unified governance. But the personal→team→company promotion is not a designed workflow — it's a collection of manual steps. A user creates a lite agent in Copilot, upgrades to Copilot Studio (one click), then manually configures sharing, permissions, and governance. There's no "promote this personal agent to a team agent" button. The concept exists in Microsoft's marketing ("standardizing how agents are shared and reused, so successful patterns move beyond individual productivity into team and enterprise workflows") but the tooling doesn't match the vision yet.
**Evidence level:** Level 1 (vendor docs describe the pieces; no practitioner evidence of this workflow working end-to-end)

### 9. Licensing confusion is a real barrier for business users
**Source:** https://www.velosio.com/blog/everything-you-need-to-know-about-microsoft-copilot-studio-licensing/ — [consultancy analysis]
**Source:** https://copilot-experts.com/microsoft-copilot-pricing-guide/ — [consultancy analysis]
**Date:** 2026
**Agent level:** All
**What:** Business users trying to build agents face confusing licensing: M365 Copilot ($30/seat) includes Copilot Studio for internal agents at "no extra cost" — but external-facing agents, advanced connectors, and heavy usage require Copilot Credits ($200/pack or pay-as-you-go). Standard connectors may or may not be included depending on license tier. Multiple consultancy guides exist specifically to explain the licensing, which is itself evidence of confusion. The forthcoming Agent 365 add-on ($15/user) adds another layer.
**Evidence level:** Level 2-3 (convergence — multiple consultancies writing explainers = market signal that licensing is confusing)

1. **A named company running a Copilot Studio agent in production for a real business process with measurable results.** This is the second research cycle and the gap persists. Dow is mentioned once in vendor marketing with no specifics. Zero independent practitioner reports of production business agents.

2. **Business user (non-developer) experience reports of building agents in Copilot Studio.** The pwmather walkthrough is the closest, but it required Power Automate (a developer/power-user skill). No "I'm in finance and I built an agent" stories exist in public.

3. **Copilot Cowork user experiences.** It's in Research Preview — no independent reviews, no practitioner reports, no "I tried it" posts. The coverage is entirely vendor announcement + general press coverage of the announcement.

4. **Personal→team agent promotion experiences.** Nobody has written about this workflow. The concept is in Microsoft's marketing but no practitioner has documented doing it.

5. **Pricing justification data.** At $30/user/month + potential Copilot Credits + E7 at $99/user, no ROI calculations from actual business users. All ROI claims come from Microsoft's own materials.

6. **Nordic-specific Copilot Studio deployments.** No Nordic company deploying business agents via Copilot Studio appeared in any search.

## Orientation

**The business agent gap is confirmed and widening.** Microsoft has the most comprehensive vision (personal→team→company, Cowork for personal, Studio for team, Agent 365 for governance, E7 for bundling). But the gap between announcement and deployment is now the defining feature of the Microsoft business agent story:

- **Copilot Cowork** (the strongest personal agent play) is in Research Preview. No users.
- **Copilot Studio** lite agents are truly accessible to business users but limited to Q&A bots. Anything useful requires developer skills (Power Automate, connectors, API configuration).
- **Pre-built agents** (HR, Finance, Sales) are embedded in Dynamics 365 — not available to general M365 customers.
- **Reliability** is a blocker. Crashes, memory loss, "practically useless" for document tasks. This is not just a feature gap — it's a trust gap.
- **Licensing** is complex enough that multiple consultancies have built business models around explaining it.
- **Agent 365** (governance) launches May 2026 — governance for agents that don't exist yet in production.

**The CTO answer for Microsoft:** Broadest platform, strongest enterprise distribution, most products — but the announcement-to-deployment gap is the largest of any vendor. Business users cannot yet build meaningful agents without developer help. The personal agent (Cowork) is not shipping. The pricing ($30-99/seat) assumes value that has not been demonstrated. Microsoft is building governance infrastructure for an agent ecosystem that hasn't proven it works for business users.

**For our platform advisory:** Microsoft is the "safe choice" that CTOs will default to — but the actual capability for business agents is thin. This creates an advisory opportunity: helping orgs understand what they can actually do TODAY with their M365 investment vs what's announced. The lite-to-Studio ceiling is the critical insight — business users will hit it fast and need developer help, which defeats the "citizen developer" promise.
