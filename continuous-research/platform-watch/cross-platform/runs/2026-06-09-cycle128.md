## Finding 1: Dynamic Workflows — Evidence Upgraded from L1 to L3

**Source:** https://news.ycombinator.com/item?id=48311705 — [practitioner direct / community discussion]
**Supporting:** https://thenewstack.io/claude-code-dynamic-workflows-test/ — [domain trade publication, June 8, 2026]
**Supporting:** https://www.infoq.com/news/2026/06/dynamic-workflows-claude-code/ — [domain trade publication, June 1, 2026]
**Date:** HN thread opened May 28–29; accessible as of June 9 (was 429 at cycle 127)
**Evidence level:** Level 3 — HN thread delivers convergent independent practitioner signals; multiple named practitioners, same pattern from different angles

**What:** The HN thread on Dynamic Workflows (previously returning 429) is now accessible. Independent practitioner reactions are converging on three distinct blockers:

1. **Correctness ceiling (most-upvoted):** "My limiting factor is not how quickly Claude can self-trudge through code. It's whether Claude is going to do the task correctly or not." Parallel throughput is irrelevant until individual-agent quality improves. Second convergent concern: "slop debt" — compounding errors from successive agent passes; "LLM clean up after an LLM doesn't work well enough to just keep it in a loop."
2. **Token cost:** One tester burned an entire monthly 5x-plan ($500+) in 18 minutes using 62 parallel Opus agents.
3. **Rate-limit errors:** ~100 "Prompt is too long" errors/day when spawning subagents at scale.

The New Stack published the first independent hands-on test (June 8, author: Jessica Wachtel): headline ran only 5 concurrent agents — far below the 1,000-agent marketing ceiling — signaling real-world usage is modest. InfoQ (June 1, Robert Krzaczyński) notes substantial token overconsumption vs. standard sessions; recommends starting with scoped small tasks.

Positive minority (HN): better automatic context management, improved code review for some tasks. Not blocking the overall blocker thesis.

**Key claims:**
- Three independent blockers confirmed: correctness ceiling, token cost, rate-limit errors
- Practical test ceiling far below 1,000-agent cap (The New Stack reached 5)
- Bun Zig→Rust flagship case criticized directly on HN as anti-pattern / "1m vibe-coded Rust lines"
- Vendor claims (76% success rate; Bun case) remain Level 0 commercial — rejected by HN practitioners
- Enterprise: Dynamic Workflows off by default, admin opt-in required (InfoQ)
- Interrupted runs auto-save and can resume (InfoQ)

**Simon Willison:** ZERO posts on Dynamic Workflows through June 9 (Day 12). Re-check: June 11. Still the clearest "notable practitioner silence" signal for this feature.

**KB update:** Dynamic Workflows evidence level: **L1 → L3**. Next milestone: June 11 (Day 14) re-check for Fortune 1000 post-mortems.
