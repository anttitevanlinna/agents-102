## Findings

### Finding 1: Salesforce Headless 360 — Umbrella Architectural Pivot

**Source:** https://salesforcedevops.net/index.php/2026/04/15/tdx-2026-reporters-notebook-salesforce-goes-headless-and-widens-the-builder-gap/ — [domain trade publication] — April 15, 2026
**Supporting:** https://www.theregister.com/2026/04/15/salesforce_headless_360/ — [tech press / bare facts] — April 15, 2026
**Date:** April 15, 2026
**Agent level:** Company
**Evidence level:** Level 1 (vendor architectural declaration; independent journalist describes it accurately but cannot verify shipping status of all 7 components)

**What:** Salesforce announced "Headless 360" at TDX — an umbrella brand for making "everything on Salesforce an API, MCP tool, or CLI command." The goal: coding agents (Claude Code, VS Code, Cursor) can access the entire Salesforce platform without ever touching a browser. Parker Harris keynote framing: "Why should you ever log into Salesforce again?" Seven coordinated sub-announcements included.

**Key claims:**
- 60+ new MCP tools and 30+ preconfigured coding skills shipping with Developer Edition
- Agentforce Experience Layer: decouples agent capabilities from UI rendering (enables multi-surface deployment)
- Agent Script open-sourced: deterministic control language now available for community use
- Agentforce Vibes 2.0: multi-model (Claude Sonnet 4.5 default, GPT-5 option), full org awareness, browser-based cloud IDE
- Free Developer Edition: 110 requests/month + 1.5M tokens/month through May 31, 2026
- AgentExchange: consolidated marketplace (AppExchange + Slack Marketplace + Agentforce) — 13,000+ listings, $50M Builders Fund

**Critical independent observation (SalesforceDevops.net):** The "builder gap" is widening. Headless 360 targets developers fluent in prompt engineering, agent orchestration, and software testing — not the traditional low-code Salesforce admin who built the ecosystem. No admin-to-agent-developer upskilling path announced. Structural constituency risk.

### Finding 2: Agentforce Grid — What It Actually Is (Beta, Not GA)

**Source:** https://admin.salesforce.com/blog/2025/salesforce-admins-streamline-ai-workflows-with-agentforce-grid — [vendor blog, Level 0] — late 2025
**Supporting:** https://smartbridge.com/agentforce-360-deep-dive-observability-grid-connected-agents-sales-in-chatgpt/ — [domain trade publication] — 2026
**Date:** Winter 2025-26 (pre-TDX description); TDX preview zone April 15-16
**Agent level:** Personal / Team
**Evidence level:** Level 0 (vendor-described capability; Beta — not GA at TDX)

**What:** Agentforce Grid is a spreadsheet-like interface inside Salesforce that chains CRM/Data Cloud queries, AI prompts, agent calls, and automated actions — bulk-processing hundreds or thousands of records without code. Described by Salesforce as "the first AI-native spreadsheet built into Salesforce." It addresses a specific pain: testing AI prompts against real data before deploying agents to production.

**Key claims:**
- Use cases: bulk CRM record enrichment, personalized outreach at scale, service transcript analysis, lead classification
- Supports batch simulations across hundreds of rows, prompt chaining, quality evaluation at scale
- **Release status: Beta** — TDX presented it in "Preview What's Next" zone, not as a GA launch
- Zero independent practitioner deployment evidence found

**KB correction:** Pre-event note flagged "unknown capability with no prior description." It was described in Salesforce Winter '26 materials — earlier searches missed the Winter '26 release context. Grid is a low-code AI workflow prototyping tool, not a new agent orchestration architecture.
