**Status clarification:** Claude Managed Agents is in **public beta**, not full GA. Multiple tech press articles confirm beta status. (Correction to Cycle 95 state.md entry which labeled it GA.)

**Architecture confirmed by multiple tech press sources:**
- Sandboxed session execution (containerized, filesystem access only to granted paths)
- Session persistence — results survive connection drops; sessions can run autonomously for hours
- Tool execution: bash commands, file operations, web search, MCP server connections

Multiple tech press sources confirm: "execution occurs exclusively on Anthropic's own infrastructure" — NOT available through Amazon Bedrock or Google Vertex AI. Sources:

**CTO implication:** This is the second Anthropic service requiring data to transit Anthropic's US infrastructure (alongside Cowork, which remains outside EU Data Boundary). ## Orientation

**The most strategically important finding this cycle is the Managed Agents multi-cloud limitation.** By running exclusively on Anthropic's own infrastructure — bypassing Bedrock and Vertex AI — Anthropic is making a deliberate choice to own the runtime layer. For an AWS-native or Azure-native enterprise, this means BOTH Cowork (outside EU Data Boundary) AND Managed Agents (Anthropic-infra-only) create data transit considerations. The moat is real: tighter model+runtime integration means better performance and faster iteration. The constraint is real: no path through existing cloud governance, no data residency guarantees through cloud partners.

**Cowork GA closes the "research preview" objection** but does not yet have independent deployment evidence beyond Simon Willison's personal-use test. The PayPal webinar (April 16) is the next data point. Need 10-20 independent enterprise reports to reach Level 3 — currently at Level 1 (Willison) + Level 0 (vendor-cited early adopters).

**The HR practitioner gap is now confirmed Level 3 by absence.** This is a research system limitation, not a deployment limitation — the publishing infrastructure for HR practitioners doesn't exist. Correct the research method: use trade press interviews and conference presentation databases instead of web search.
