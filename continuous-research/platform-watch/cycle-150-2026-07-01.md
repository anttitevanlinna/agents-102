**[Finding 2.1] arXiv 2606.28791 — "From Determinism to Delegation" (Alenezi)**
Source: https://arxiv.org/abs/2606.28791 — [academic/research]
Date: June 27, 2026
What: Paper argues software engineering value is shifting "from writing deterministic code to supervising probabilistic and autonomous behavior." Identifies three shifts: (1) unit of work: functions → supervised agent workflows; (2) correctness: binary assertions → statistical evaluation under uncertainty; (3) accountability: code authorship → outcome ownership. Concludes "symbiosis rather than substitution" and "disciplined oversight rather than automation as the critical competency."
Evidence level: L1 (single academic paper, theoretical framing; empirical evidence noted as "mixed")
Key claims: "Engineering value is shifting from writing deterministic code to supervising probabilistic and autonomous behavior." Notes that some studies show productivity gains, others document slowdowns among experienced developers.

**[Finding 4.1] "Compiled AI" arXiv paper — deterministic execution approach**
Source: https://arxiv.org/abs/2604.05150 — [academic/research]
Date: April 6, 2026
What: "Compiled AI: Deterministic Code Generation for LLM-Based Workflow Automation" — LLMs generate executable code artifacts during a compilation phase; workflows then execute deterministically without further model invocation. Specifically targets high-stakes enterprise workflows (healthcare focus). Constrains generation to narrow business-logic functions embedded in validated templates.
Evidence level: L2 (single academic paper proposing an approach)
Key claims: "Non-determinism persists even at temperature=0, undermining reproducibility in interpreted AI approaches." Analysis of 1,600+ annotated traces across seven major frameworks: specification failures (41.8%) and inter-agent coordination failures (36.9%) = 79% of failures are NOT infrastructure problems. Survey of 306 practitioners across 26 domains: 68% limit agents to ≤10 autonomous steps before human intervention, reliability cited as primary deployment barrier.
Relevance: Converges on the same pattern (determinism as the production goal) from an independent academic starting point.

## META-LEARNING

1. **arXiv ID accuracy is critical.** When a cycle-to-cycle briefing cites a specific arXiv ID, verify the ID matches the described paper before treating it as established. 2606.29108 mismatch (Symbolon vs. n8n study) would have contaminated the convergence analysis if not caught.

2. **Prediction market snapshots age fast.** Polymarket July 1 at 35% (June 28) was wrong by June 30 — the political track resolved before traders priced it. Markets price the *expected* path, not the actual path when geopolitical negotiations are involved.

3. **Status page lag is a distinct signal.** status.claude.com still showing suspension as "Monitoring" while the newsroom announces restoration is not a contradiction — it's a display lag. Distinguishing lag from contradiction matters when synthesizing multi-source status.

4. **"Person-first" searches beat topic searches for finding practitioner reactions.** Searching "simonwillison.net July 2026" directly was more efficient than searching "n8n study 2026 practitioner reaction." The latter returned only the papers themselves; the former confirmed absence within that person's known publication location.
