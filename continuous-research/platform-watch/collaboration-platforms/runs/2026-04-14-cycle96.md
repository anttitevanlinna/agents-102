## Findings

### Finding 1: Agents in Jira GA — governance-first design signal (Feb 25, 2026)

**What shipped:** Atlassian launched Agents in Jira (open beta Feb 25, 2026), bringing Rovo agents directly into Jira workflows. Agents can receive task assignments, collaborate via comments, and execute workflow steps. All actions captured in work item history alongside human activity. OAuth 2.1 authentication, audit logging, and RBAC — agents operate inside Jira's existing permission structures, not alongside them.

**The governance signal:** Atlassian SVP Sanchan Saxena explicitly acknowledged the risk: "What we don't want is agents that do 10 times the work but also create 10 times the chaos." The design decision to tie agents to existing permissions rather than create parallel permissions is a deliberate architectural choice. Agents have no permission to delete or modify production code. Human approval required before changes become permanent.

**MCP adoption data (strongest stat this cycle):** Enterprises account for nearly 50% of all Rovo MCP Server usage, and customers on paid Atlassian editions drive 93% of that activity. Nearly one-third of MCP operations are writes (creating/updating records), not just reads. This is independently verifiable enterprise behavior, not an announcement.

**Source:** [TechInformed: Atlassian adds AI agents to Jira under existing permissions](https://techinformed.com/atlassian-adds-ai-agents-to-jira-under-existing-permissions/) [domain trade publication] — Feb 25, 2026
**Evidence level:** Level 2 (documented deployment behavior with independently verifiable MCP usage stats)

### Finding 2: Agent sprawl is now a documented governance crisis — not hypothetical

**What's documented:** The agent sprawl problem our previous cycle predicted is now empirically documented:
- More than 3 million AI agents operating within corporations, only 47.1% actively monitored or secured, leaving an estimated 1.5 million agents running without oversight (Gravitee State of AI Agent Security 2026, n=900+)
- 94% of organizations report concern that AI sprawl is increasing complexity, technical debt, and security risk (OutSystems, reported via PRNewswire)
- More than 80% of IT leaders believe proliferating AI agents will create more complexity than value due to integration challenges and silos (2026 Salesforce Connectivity Benchmark Report, n=1,050 IT leaders, conducted with Vanson Bourne and Deloitte Digital)
- Only 36% have a centralized approach to agentic AI governance; only 12% use a centralized platform to maintain control
- Companies currently average 12 agents, expected to reach 20 by 2027

**The collaboration platform connection:** The sprawl problem is being driven by the exact "Trojan horse" dynamic we theorized — agents arriving through every tool people already use, with no unified governance. Collaboration platforms are the primary vector.

**New governance market forming:** AWS Agent Registry (announced April 13, 2026) gives enterprises visibility into every agent running across their estate, metadata for all agents/tools/MCP servers. Microsoft Agent 365 + Entra Agent ID. Google Cloud API Registry + Vertex AI Agent Builder. ServiceNow AI Control Tower. All major platforms are now building agent registries — a direct response to sprawl.

**Source:** [CIO Dive: IT leaders grapple with AI agent sprawl](https://www.ciodive.com/news/it-leaders-grapple-ai-agent-sprawl-integration/811411/) [domain trade publication]; [OutSystems/PRNewswire: 94% sprawl concern](https://www.prnewswire.com/apac/news-releases/agentic-ai-goes-mainstream-in-the-enterprise-but-94-raise-concern-about-sprawl-outsystems-research-finds-302739251.html) [general press — stat only]; [thelettertwo.com: AWS Agent Registry](https://thelettertwo.com/2026/04/09/aws-unveils-agent-registry-to-bring-order-to-enterprise-ai-sprawl) [domain trade publication] — April 9, 2026
**Evidence level:** Level 3 (multiple independent surveys converging on the same governance gap)

### Finding 3: Notion Custom Agents — two named enterprise production deployments, first real agentic evidence

**What's running in production:**

**Remote (global HR company):** IT team replaced help desk system with Custom Agents. Agents triage with >95% accuracy and autonomously resolve 25%+ of tickets. Saves IT team 20 hours per week. Keeps Slack and Notion in sync.

**Ramp ($32B fintech):** Over 300 agents in production. The "Product Oracle" answers dozens of questions daily about the roadmap and how features work (Slack-connected, Q&A). "Sales Feedback Categorizer" maps inbound sales feedback to the product roadmap. "Referral Bonus Roy" identifies customers who should receive a referral bonus and routes work to completion. Ramp cut productivity-tool costs roughly 70% and helped teams move 3x faster. In mid-2025, Ramp started experimenting with autonomous Notion Agents.

**Scale:** 21,000+ agents built in beta. Notion internally runs 2,800 agents (more than the company has employees).

**Critical caveat:** Both Remote and Ramp cases are surfaced via Notion's own customer stories and launch blog. These are curated success cases. Remote's "25% autonomous" claim and Ramp's "300 agents" are from the vendor's launch materials, citing the customer — but not independently verified. Classified as Level 0 proximity (vendor-curated), Level 2 in substance because named companies + named practices + measurable outcomes.

**What doesn't work (practitioner-sourced):** Agents cannot create other agents (manual setup required for each). When editing one agent, locked to that agent (no cross-agent bulk editing). No audit trail infrastructure for regulated industries — explicitly flagged as unaddressed. Workspace-confined for writes (can read external, cannot write back to most external systems except via MCP). Slack Enterprise Grid not supported.

**Source:** [Notion: Introducing Custom Agents](https://www.notion.com/blog/introducing-custom-agents) [vendor press release — Level 0, sourcing remote/ramp claims]; [buildtolaunch.substack.com: Notion AI Agents limitations guide](https://buildtolaunch.substack.com/p/notion-ai-agents-examples-guide) [practitioner direct] — Feb/March 2026; [Creator Economy: Ramp agents](https://creatoreconomy.so/p/inside-ramp-the-32b-company-ai-agents-geoff-charles) [practitioner direct — paywall limited]
**Evidence level:** Level 2 (vendor-curated named deployments; practitioner limitations independently sourced)

### Finding 4: Rovo — backend LLM swap broke production agents (March 31, 2026)

This is the highest-signal counter-evidence finding for this cycle.

On March 31, 2026, Atlassian migrated all Rovo Agents to a new backend LLM model. The migration broke agents that relied on attachment analysis — agents could no longer access files in the primary Attachments field, only through comments/activity. Community posts express urgency about teams' production workflows breaking.

Additionally: Rovo agents still cannot handle custom Jira fields (a "huge limitation for many teams"). Context degradation reported as projects grow — "occasionally losing track of important context or losing consistency in following instructions."

**The pricing trap is landing:** Consumption-based pricing transition in early 2026 has teams worried about cost predictability after adoption lock-in. Enterprise users still get ~7 "engagements" per month (extremely tight for production use). Community concern is real.

**What works well:** Within-Atlassian workflows work cleanly. MCP Server is genuinely enabling external AI clients (Claude, Cursor, VS Code, GitHub, Docker, Figma, and 15+ others) to read and write Jira/Confluence. Enterprise adoption of MCP is real (50% of usage, 93% from paid editions).

**Source:** [Atlassian Community: Backend LLM model update](https://community.atlassian.com/forums/Atlassian-AI-Rovo-articles/Important-Update-Rovo-Agents-are-getting-a-new-backend-LLM-model/ba-p/3207356) [practitioner direct — community posts] — March 2026; [eesel.ai: Rovo review](https://www.eesel.ai/blog/rovo) [practitioner analysis]
**Evidence level:** Level 2 (practitioner-reported production failures, community-confirmed)

### Finding 5: Agentforce in Slack — ROI unproven beyond Salesforce itself

**Analyst finding (Keith Kirkpatrick, Futurum):** "The larger question around Agentforce is whether these productivity data points will ultimately result in top- or bottom-line benefits to the business." Time savings alone (the 500K hours/year Salesforce claims for internal use) don't guarantee business value unless capacity is redirected to revenue-generating activities. Salesforce's figures rely on "experience-driven evidence rather than independently verified, long-term adoption metrics from neutral sources."

**What's confirmed:** Agentforce ARR hit $800M, up 169% YoY. 29,000 deals closed, up 50% QoQ in Q4. These are financial metrics and are reliable. 2.4 billion "agentic work units" delivered in Agentforce and Slack — but "work units" is vendor-defined.

**External customer evidence (thin):** Accenture (Salesforce Business Group) uses Agentforce for contact/close plan creation, surface insights, dynamic updates — cited in Salesforce's own deployment blog. No independent Accenture source. Still vendor-curated.

**The Customer Zero problem:** 86% Salesforce employee adoption is Salesforce using their own product. Even Salesforce acknowledges "success depends as much on change management as technical implementation." No independent enterprise has published deployment results.

### Finding 6: Zoom AI Companion 3.0 — language degradation for non-English enterprise

**New limitation surfaced:** AI Companion's summarization quality degrades significantly with non-English content. Teams conducting mixed-language conversations report substantially reduced accuracy and utility. ### Finding 7: MCP + A2A protocol stack becoming enterprise consensus

**What's emerged:** The three-layer AI protocol stack is becoming the consensus enterprise architecture:
- **MCP** — tools and data context (Atlassian, Slack, Notion all MCP-enabled)
- **A2A (Agent-to-Agent)** — agent coordination layer (Linux Foundation hosted, 150+ organizations supporting, Google/Microsoft/AWS integrated)
- **WebMCP** — web access layer

**A2A milestone:** One year in, A2A Protocol has 150+ supporting organizations, deep integration across Google, Microsoft, and AWS, and active production deployments across supply chain, financial services, insurance, and IT operations. Collaboration platforms have not yet announced A2A adoption — this is the next frontier.

**Implication for Trojan horse thesis:** If collaboration tools adopt A2A, agents in Jira could coordinate with agents in Slack could coordinate with agents in Notion — making the "workspace confinement" problem solvable at protocol level rather than requiring a single-vendor platform. This would change the thesis significantly.

**Source:** [A2A Protocol: Adoption milestones](https://zexprwire.com/a2a-protocol-surpasses-150-organizations-lands-in-major-cloud-platforms-and-sees-enterprise-production-use-in-first-year/) [general press — facts only]; [DEV Community: MCP vs A2A guide](https://dev.to/pockit_tools/mcp-vs-a2a-the-complete-guide-to-ai-agent-protocols-in-2026-30li) [practitioner analysis] — 2026
**Evidence level:** Level 2 (documented protocol adoption; collaboration platform A2A adoption not yet confirmed)

## What I Looked For But Did Not Find

1. **Independent (non-vendor) enterprise deployment evidence for Agentforce in Slack.** After 6+ targeted searches, zero independent enterprise has published Agentforce deployment results. Customer Zero (Salesforce itself) is the only documented case. Every other "deployment" story traces to Salesforce-curated sources.

2. **Nordic company deploying collaboration platform agents.** Zero Nordic-specific evidence found despite targeted searches. The Zoom non-English degradation finding is Nordic-relevant (Finnish/Swedish meetings) but not Nordic-origin.

3. **Practitioner accounts of cross-system orchestration through collaboration tools.** MCP enables external AI to reach into Jira/Confluence, but no practitioner has published a documented workflow where a Rovo agent actually read from AND wrote to three or more external systems autonomously. All evidence is either "can read" or "can write Jira" — not multi-system orchestration.

4. **Governance framework for agents in collaboration tools.** Atlassian is closest (existing permissions + audit logs). No platform offers Cedar-style policy enforcement, formal verification, or agent identity management comparable to what Microsoft Entra Agent ID offers. The market is building registries (AWS, Microsoft) but policy enforcement is not yet there.

5. **Any security incident from agents with write access.** The "when not if" moment has not been publicly documented yet. No named incident of a Rovo or Notion agent creating a production mess. Absence is not reassurance — it may reflect that these deployments are too recent.

6. **Concrete evidence of the pricing cliff impact.** Notion's credit pricing kicked in May 3, 2026 (after this cycle). Rovo's consumption shift is in progress. No documented case of a team abandoning collaboration platform agents when pricing changed.

## Orientation

### Does the Trojan Horse thesis hold? (Updated assessment)

**Yes, and the urgency has increased.** The thesis holds stronger than the previous cycle:

1. **Trojan horse is confirmed, not hypothetical.** 94% of enterprises report sprawl concern. 3M+ agents running, 1.5M unsecured. Collaboration tools (Jira, Slack, Notion, Zoom) are the primary vector because they require no procurement decision — they arrive as feature toggles.

2. **The wooden horse metaphor holds but needs refinement.** The previous cycle's verdict ("carries co-pilots, not agents") needs updating for Notion and Atlassian specifically:
   - **Notion Custom Agents** now run genuine autonomous work (Remote's 25% autonomous ticket resolution, Ramp's 300+ production agents). These pass the "multi-step autonomous" test within the workspace.
   - **Rovo MCP Server** with 93% enterprise adoption from paid editions and 1/3 write operations is evidence of real production use, not just browsing.
   - **The wooden payload is still in Zoom** — personal workflow automation, not autonomous reasoning.

3. **Agent sprawl is the realized risk, not a prediction.** The governance void that the previous cycle flagged as coming is now documented: 1.5 million unsecured enterprise agents. Every major cloud provider building agent registries is not a coincidence — it's response to production reality.

4. **The platform choice doesn't disappear — it escalates.** The question has shifted from "which platform?" to "how do we govern what's already arrived?" Atlassian's governance-first design (agents inside existing permissions) is the most mature answer in this category. But even Atlassian acknowledges the chaos risk.

5. **A2A is the critical wildcard.** If collaboration platforms adopt A2A protocol, cross-tool agent coordination becomes architecturally possible without a single-vendor platform. This could make the collaboration tool layer a genuine enterprise agent platform rather than a collection of silos. Currently: not adopted by collaboration platforms. Watch item for next cycle.

### The CTO insight this cycle adds:

The Trojan horse has already entered the gates. The question is not whether to allow it — 80%+ of IT leaders are already dealing with agent proliferation driven by collaboration tool feature toggles. The question is whether to govern the agents already running or wait for a "platform strategy" moment that may never arrive cleanly. The enterprises that win will be the ones that treat collaboration platform agents as the first wave of their agent program rather than the shadow IT they haven't gotten around to addressing.

**Rovo's governance-first design (agents inside existing permissions, audit trail, human approval for high-impact actions) is the best available template for this problem.** It's not perfect, but it's the most thoughtful approach in the collaboration platform space.
