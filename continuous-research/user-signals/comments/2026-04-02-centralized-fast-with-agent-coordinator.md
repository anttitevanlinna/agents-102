**Observation:** Centralized CAN work fast — if the coordinator is an agent, not a human. The bottleneck in centralized models is human processing speed at the center (IT backlog, approval chains, architecture reviews). Replace the human coordinator with an agent coordinator and the speed problem disappears. This means the centralized vs. distributed debate is really about: where does governance live? Not about speed. Both models can be fast. The hybrid: agent orchestrator (centralized, fast) for cross-boundary coordination + personal agents (distributed, fast) for domain execution + shared doctrine (skill files) as connective tissue. Mirrors Moltke: general staff sets doctrine, units execute autonomously.

**Gap:** No platform is building this hybrid model. Centralized platforms (Frontier, Agent 365) assume human governance. Distributed approaches assume no coordinator. The "agent general staff" pattern — an agent that coordinates other agents via doctrine, not commands — is untracked.
