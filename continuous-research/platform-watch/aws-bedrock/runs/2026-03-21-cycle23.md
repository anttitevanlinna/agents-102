2. **Independent enterprise deployment evidence.** No named company reports production business agent deployment on AgentCore with measurable outcomes.

4. **Multi-system orchestration evidence.** AgentCore supports MCP/A2A/AG-UI protocols, but no evidence of anyone orchestrating across 5+ business systems using it.

## Orientation

**AgentCore is the most architecturally complete managed agent runtime** — supporting all three protocols (MCP + A2A + AG-UI), Cedar-based governance, and stateful sessions. But it's pre-adoption: zero independent evidence, Python-only SDK, no IaC, runtime reliability issues, and a security vulnerability in its starter toolkit. The Cedar-based Policy GA is genuinely novel — no other runtime offers formal-verification-backed agent governance. But the SailPoint partnership (Level 0) confirms that identity governance is not yet built-in.

**For AWS-native companies**, AgentCore is the natural choice for agent runtime — but it requires a team that can build Docker images and push to ECR. For business users, it offers nothing directly. This is pure infrastructure, not a business agent surface.

**The AG-UI correction is important:** We previously attributed AG-UI to AWS. It's CopilotKit's protocol, broadly adopted. AWS is an implementor, not the creator. This strengthens the three-protocol stack thesis (MCP + A2A + AG-UI) but weakens any AWS-specific moat claim.
