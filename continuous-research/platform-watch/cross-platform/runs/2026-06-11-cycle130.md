### Finding 1: Kiro = AWS Agentic IDE — Spec-Driven Development (Level 2)

**Source:** [kiro.dev](https://kiro.dev/) — [vendor press release]; [lushbinary.com Jun 2026](https://lushbinary.com/blog/ai-coding-agents-comparison-cursor-windsurf-claude-copilot-kiro-2026/) — [domain trade publication]; [petermcaree.com Dec 18 2025](https://petermcaree.com/posts/kiro-agentic-ide-hype-hope-and-hard-truths/) — [practitioner direct]; [dev.to/fikuri Sep 13 2025](https://dev.to/fikuri/kiro-the-good-bad-and-ugly-part-in-my-personal-experience-1neh) — [practitioner direct]; [chatforest.com May 23 2026](https://chatforest.com/reviews/amazon-kiro-aws-agentic-ide-spec-driven-review/) — [practitioner analysis]; [GeekWire 2026](https://www.geekwire.com/2026/aws-targets-ai-slop-with-new-spec-check-in-kiro-coding-tool-amid-scrutiny-of-agent-reliability/) — [general press]
**Date:** Public preview summer 2025; international launch May 7, 2026
**Agent level:** personal / team
**What:** Kiro is Amazon Web Services' agentic IDE — a replacement for Amazon Q Developer built on Code OSS (VS Code fork). Core differentiator is spec-driven development: prompts are converted to structured markdown files (requirements.md, design.md, tasks.md using EARS notation), developer reviews and approves the spec, code is generated from the spec. Explicitly positioned against "vibe coding" (freeform prompt → code). Two independent practitioners (McCaree Dec 2025, fikuri Sep 2025) tested it over multiple weeks with production use cases.
**Evidence level:** Level 2 (two independent hands-on experiments). May 2026 features (parallel tasks, spec-check) remain Level 0 (vendor claims only).
**Key claims:**
- Made by AWS; official docs at aws.amazon.com/documentation-overview/kiro/; product at kiro.dev
- Model routing: Claude Sonnet 4.6 (reasoning) + Amazon Nova (high-throughput code generation) + open-weight options (Qwen3, DeepSeek v3.2)
- Hooks: event-driven automation on file save, tool use, task completion — claimed unique in the category
- May 2026 parallel task execution: independent tasks run concurrently; spec-check (mathematical proof requirements are contradiction-free) per GeekWire
- Pricing: Free (50 credits/mo) / Pro $20/mo / Pro+ $40/mo / Power $200/mo; overage $0.04/credit
- AWS ecosystem integration: Aurora DSQL, Amazon Connect, AWS Transform agents (April 2026)
- Independent practitioners: strengths = spec methodology improves planning discipline, Claude Sonnet handles long context well. Weaknesses = pricing opaque/punishing for small tasks, spec workflow overkill for minor changes, terminal hangs, hooks limited to explicit IDE saves
- Feb 2026 outage incorrectly attributed to Kiro by some — Amazon denied ([The Register, Feb 2026](https://www.theregister.com/2026/02/20/amazon_denies_kiro_agentic_ai_behind_outage/))
