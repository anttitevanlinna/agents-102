### Finding 3: Three Breaking API Changes + Hidden Tokenizer Cost Shift
**Source:** https://chatforest.com/builders-log/claude-sonnet-5-launch-agentic-upgrade-migration-guide-builder-decision/ — [practitioner analysis]
**Date:** June 30, 2026
**Agent level:** team/company (API integration)
**What:** ChatForest (practitioner-level migration guide) documents three breaking changes for existing Claude API integrations: (1) adaptive thinking enabled by default, (2) sampling parameters restricted (temperature/top_p/top_k return 400 error if non-default), (3) manual extended thinking syntax removed. Additionally, the new tokenizer produces ~30% more tokens for identical input text — a hidden cost shift that inflates costs before any pricing change takes effect. Builders on AWS Bedrock legacy InvokeModel API must migrate to Claude Platform to access Sonnet 5.
**Evidence level:** Level 2 (practitioner analysis with specific API details; migration guide is verifiable against API docs)
**Key claims:**
- Adaptive thinking: enabled by default; consumes token budget unless explicitly disabled with `thinking: {type: "disabled"}`
- Sampling parameters: `temperature`, `top_p`, `top_k` set to non-default values now return HTTP 400 error
- Extended thinking: deprecated `{type: "enabled", budget_tokens: N}` syntax removed; must use `effort` parameter
- Tokenizer: ~30% more tokens for identical input vs Sonnet 4.6 — at $2/MTok intro price, a $1 prompt becomes ~$0.87; after Aug 31 at $3/MTok, same prompt costs $1.30
- 1M context window effectively holds only ~770K token-equivalents of Sonnet 4.6 content
- AWS Bedrock users on legacy InvokeModel API must migrate
