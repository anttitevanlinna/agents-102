**Observation:** The enterprise software mental model (one system per function, centralized, governed top-down) may be the wrong model for agents. Distributed autonomous agents with shared doctrine — like Moltke's Auftragstaktik — may be structurally superior. Reasons: no single point of failure, no capability ceiling, faster evolution, context stays local, learning is parallel. Governance comes from shared skill files (doctrine), not a central platform. Maps to the competence-first sequence: competence → personal agents → coordination → governance emerges from practice. Centralized agent platforms (Frontier, Agent 365) may be solving the wrong problem — like building one general to command every soldier, when the real advantage is trained officers acting on shared intent.

**Gap:** Research assumes centralized platform is the target state. No investigation of distributed agent coordination in enterprise. Need to track: who is running multiple autonomous agents that coordinate? Is CLAUDE.md / skill.md functioning as "agent doctrine"? Does the Moltke/Auftragstaktik parallel hold under enterprise conditions?
