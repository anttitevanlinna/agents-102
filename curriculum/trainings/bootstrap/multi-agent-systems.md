# Multi-Agent Systems

## Big Idea
One challenge, three angles. Each agent sees a slice the others miss.

## Meta
- **Primary Bloom's level:** Analyze → Create
- **Prework:** agent memory (filesystem as memory, how agents carry context between runs); agent tools
- **Homework:** selected "What is an Agent" sections (multi-agent coordination)
- **Materials (trainer):** three-agent scaffold (roles, handoff format), sample synthesis prompt. The exercise runs on the participant's Module 2 challenge brain — no separate data pre-load.
- **Plug points:** the participant's challenge brain from Module 2

## What You'll Learn
After this module, you will be able to:
- **Design** an assembly-line agent pipeline: three agents, sequential handoffs
- **Build** three specialised agents, each looking at your challenge through a different lens
- **Analyze** handoff failure modes — noise, bias, mis-framing at the seams
- **Synthesize** cross-agent findings that no single agent could produce
- **Evaluate** when a single agent beats three

## Lectures
- **Setting Up Proper Multi-Angle Analysis** — what works, what doesn't, what breaks. Synthesis techniques: turning three agents' takes into findings that hold up.

## Exercises
- **Three lenses on your challenge** — Build three specialised agents that each interrogate your Module 2 challenge brain from a different angle: *the risk lens* ("what's the biggest risk I'm not seeing?"), *the stakeholder lens* ("who haven't I engaged yet, and why does that matter?"), *the cheap-experiment lens* ("what's the smallest thing I could run this week to learn something real?"). Each agent reads the same brain but with a different instruction set. Run each for two rounds — round two sharpens as the agent notices what it missed the first time. Synthesize across all three. The finding no single agent could produce: the risk you can only see once the stakeholder lens names who's been left out, and the experiment cheap enough to survive that risk.

## Key Concepts (Emergent)
- **Assembly line**, not a mesh. Sequential hand-offs before orchestration.
- **Seams are where it fails.** Multi-agent coordination is a handoff problem.
- **Three agents here because they ask genuinely different questions** — not because splitting is inherently better. Anthropic's warning: only a few situations where multi-agent wins.
