# Crux — diagnose before you prescribe

Richard Rumelt's *Good Strategy Bad Strategy*. Most strategy is a list of goals with a motivational verb attached. Real strategy starts with an plain diagnosis: what is actually in the way? Out of everything in the way, which single obstacle — if it moved — would release pressure on the rest? That one is the crux.

This skill forces the sequence. Diagnosis, then guiding policy, then coherent actions. In that order.

**Premature prescription without diagnosis is the failure mode this skill exists to prevent.** If the user hands you a problem and asks for recommendations, refuse the shortcut. Go through the steps.

## Input

One of:
- A memory folder (notes, source files, observations about a situation)
- A challenge brief written in prose
- A messy set of bullets from a meeting

If the input is thin, ask two questions maximum: *What's the situation?* and *Who is affected?* Then proceed.

## Process

1. **List every obstacle.** Read the input carefully. Name everything standing between the current state and the desired state. Don't filter yet — 12 to 20 items is normal. Structural, political, technical, human, temporal, financial. Write them as short sentences, not abstract categories.

2. **Cluster into 3 to 5 groups.** Name each cluster by what the obstacles share, not by the department that owns them. Good cluster names describe a mechanism ("we don't know what good looks like"), not a topic ("data quality").

3. **Find the crux.** Ask of each cluster: *if this one cluster moved, what happens to the others?* The crux is the cluster whose resolution releases pressure on the most other clusters. Name it. Defend the choice in two or three sentences. If two clusters look equally load-bearing, say so and pick the one that is more addressable this quarter — strategy has to be acted on.

4. **Draft a guiding policy.** One paragraph. The guiding policy is not a goal ("we will become agent-ready"). It is an approach that tells you what to do when new situations come up ("we will build agent competence in the five processes where rules are already written down, and refuse platform commitments until that competence exists"). It must address the crux head-on.

5. **Name the hard choice.** What does the guiding policy *refuse*? Rumelt's test: a guiding policy that doesn't close off any option isn't a policy, it's a wish. State one or two things the policy deliberately does not do.

6. **List three coherent actions.** Not a to-do list. Three moves that the guiding policy makes obvious — each one reinforces the others, each one is something someone in the room can start on Monday.

## Output

```
Obstacles (ordered, most load-bearing first)
1. ...
2. ...

Clusters
A. [name] — [obstacles it contains]
B. ...

The crux
[Name the cluster. Defend in 2-3 sentences — why this one releases pressure on the others.]

Guiding policy
[One paragraph. Addresses the crux. Tells a new situation what to do.]

What this policy refuses
- [Hard choice 1]
- [Hard choice 2]

Three coherent actions
1. ...
2. ...
3. ...
```

## Notes

- If the user pushes back on the crux, reconsider — they know the terrain. But don't switch just because the crux is uncomfortable. Uncomfortable is often correct.
- A guiding policy that could survive on any other company's slide is too abstract. Put the specific domain in the words.
- Stop at three actions. Five feels thorough and signals you haven't chosen.
