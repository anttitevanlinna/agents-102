**Observation:** "Team AI" and "Company AI" may not need separate infrastructure. Most company-level processes are buildable with personal agents that share skills and communicate. There is no "company-human" — companies run on individual humans coordinating. Agent organizations should follow the same pattern: personal agents + shared skill files + cross-agent messaging + governance norms. The 25 governance tools are solving the wrong problem — building the org chart for agents when agents, like humans, don't work through org charts. Bottom-up (personal agents learning to coordinate) may beat top-down (governance platforms imposing structure).

**Gap:** No research tracks the "personal agents coordinating as team/company" pattern. All platform research assumes team/company agents are separate, harder problems requiring new infrastructure. Need to investigate: who is doing bottom-up agent coordination today? Is shared CLAUDE.md / skill.md already functioning as team-level agent capability?
