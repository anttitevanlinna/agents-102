### 2. Agent Skills Becomes Cross-Platform Standard — OpenAI Quietly Adopted It
**Source:** https://venturebeat.com/technology/anthropic-launches-enterprise-agent-skills-and-opens-the-standard — [tech press]
**Also:** https://thenewstack.io/agent-skills-anthropics-next-bid-to-define-ai-standards/ — [domain trade publication]
**Also:** https://www.pulsemcp.com/posts/openai-agent-skills-anthropic-donates-mcp-gpt-5-2-image-1-5 — [domain trade publication]
**Also:** https://siliconangle.com/2025/12/18/anthropic-makes-agent-skills-open-standard/ — [tech press]
**Date:** December 2025 launch, adoption confirmed through March 2026
**What:** Anthropic published Agent Skills as an open standard (agentskills.io) in December 2025. Skills are directories with SKILL.md metadata that agents discover and load dynamically ("progressive disclosure"). Microsoft adopted it in VS Code and GitHub. OpenAI adopted structurally identical architecture in ChatGPT and Codex CLI. Canva, Stripe, Notion, Zapier built partner skills at launch. Standard donated to AAIF.
**Evidence level:** Level 3 (convergence — multiple competing platforms adopting the same standard)
**Key claims:**
- Skills are portable across Claude Code, OpenAI Codex, Cursor, and any adopting platform
- OpenAI's adoption was initially quiet, then officially announced for Codex
- Analyst skepticism exists: Holger Mueller (Constellation) notes single vendors rarely set standards alone
**Why it matters:** Anthropic is running the MCP playbook again — open the standard, get competitors to adopt, win by being the infrastructure layer. If Agent Skills achieves MCP-level adoption, "skill portability" becomes a real enterprise procurement argument.

### 4. Agent Identity Crisis Confirmed at Scale — NIST Responding
**Source:** https://www.iansresearch.com/resources/all-blogs/post/security-blog/2026/02/24/ai-agents-are-creating-an-identity-security-crisis-in-2026 — [domain trade publication]
**Also:** https://www.pillsburylaw.com/en/news-and-insights/nist-ai-agent-standards.html — [domain trade publication]
**Also:** https://securityboulevard.com/2026/03/rsac-2026-innovation-sandbox-geordie-ai-architect-of-enterprise-ai-agent-security-governance-systems/ — [domain trade publication]
**Also:** https://neuraltrust.ai/blog/rbac-ai-agents — [practitioner analysis]
**Date:** February-March 2026
**What:** Multiple independent sources confirm agent identity/security is reaching crisis status. Key data points: non-human identities exceed human users 50:1 in enterprises (projected 80:1 within two years). Only 24.4% of organizations have full visibility into agent-to-agent communication. Only 52% can track all data accessed by agents. NIST launched AI Agent Standards Initiative (CAISI) in January-February 2026, publishing concept paper on software and AI agent identity/authorization. Geordie AI appeared at RSAC 2026 Innovation Sandbox specifically for enterprise agent security governance. Traditional RBAC breaks down for agents because agents are dynamic, context-dependent, and operate at machine speed.
**Evidence level:** Level 3 (convergence — multiple independent security researchers, NIST, and startups all pointing at the same problem)
**Key claims:**
- NHI-to-human ratio: 50:1 current, 80:1 projected
- 24.4% have full agent-to-agent visibility (Gravitee survey)
- 82% of executives *think* their policies protect against agent misuse — massive confidence-reality gap
- NIST formalizing expectations for agent authentication, authorization, governance
- Prediction: high-profile data breaches from over-permissioned agents in 2026
**Why it matters:** This validates our "93% unscoped API keys" finding and escalates it. The confidence-reality gap (82% of executives think they're covered, while only 24% have actual visibility) is the single most dangerous number in enterprise AI right now. This is a core curriculum point for Module 4 (Security).

## Orientation

### What changed since Cycle 24

1. **Standards convergence is now institutional.** AAIF with 146 members governing MCP + AGENTS.md + goose means the "which standard?" debate is settled for tool connectivity. The open question is now governance *of* the standard, not *which* standard. Agent Skills is following the same trajectory 3 months behind MCP.

2. **The governance product layer is emerging.** ConductorOne (MCP governance), Agent 365 (Microsoft agent orchestration), Geordie AI (agent security), Veza + ServiceNow (identity control plane) — these are the first products specifically built for "how do we control agents in production." Six months ago this layer didn't exist. This is the most important market development for enterprise buyers.

3. **The agent identity crisis is now quantified.** 50:1 NHI-to-human ratio, 24.4% visibility, 82% executive overconfidence. These numbers make the security case concrete. NIST responding means regulatory pressure is coming.

4. **The failure data is devastating — and useful.** 95% failure rate (MIT), 40% cancellation (Gartner), 6% high performers (McKinsey). The gap between vendor ROI claims (171%) and actual outcomes (95% fail) is the single most important data point for our positioning. We exist in that gap.

### Updated cross-platform picture

The enterprise agent stack is crystallizing into four layers:
1. **Models** — multi-model is now default (Microsoft running Claude + OpenAI proves it)
2. **Protocols** — MCP (tools) + A2A (agents) + Agent Skills (capabilities), all under AAIF
3. **Governance** — Agent 365, ConductorOne, Geordie AI, Veza — the layer that just started existing
4. **Applications** — Copilot Cowork, Claude Cowork, Salesforce Agentforce, etc.

The gap is between layers 2 and 3. Protocols exist; governance products are just shipping. This is where enterprises get hurt (95% failure rate) and where our training creates the most value.
