### Finding 9: "From Determinism to Delegation" — Agentic Engineer Archetype Documented (arXiv June 27)
**Source:** [arXiv 2606.28791 June 27 2026](https://arxiv.org/abs/2606.28791) — academic
**Date:** June 27, 2026
**Agent level:** team
**What:** Mamdouh Alenezi (academic) identifies three core shifts in AI-native engineering: (1) unit of work from functions to supervised agent workflows, (2) correctness from binary assertions to statistical evaluation under uncertainty, (3) accountability from code authorship to outcome ownership. New "agentic engineer" archetype: primary task is supervising probabilistic autonomous behavior, not writing deterministic code. Key competencies: reasoning-acting loops, context engineering, tool use, memory management, behavioral drift monitoring, compositional error containment. Mixed productivity evidence noted: experienced developers sometimes experience slowdowns during transition. Concludes "symbiosis rather than substitution."
**Evidence level:** Level 1 (academic paper, single author — academic corroboration of practitioner convergence from Osmani/Ronacher/Fung)
**Key claims:**
- Three-shift model: unit/correctness/accountability all move from deterministic to probabilistic/delegated
- "Agentic engineer" = supervisor of autonomous behavior, not code author
- Experienced developer slowdowns during transition documented
- Directly corroborates Osmani (cycle 143): "hard part moved from writing code to deciding whether to trust it"

### Finding 10: N8n Agentic Workflow Study — LLMs Embedded in Deterministic Pipelines (arXiv June 27)

- Reliability mechanisms uncommon in production
- Contradicts vendor narratives about autonomous agents — real deployments are hybrid LLM-in-pipeline
- Confirms KB pattern: hybrid beats autonomous (Pattern X)

### Finding 11: SWE-Interact — Models Score ~25% on Interactive Variant vs. ~50% Single-Turn (arXiv June 29)
**Source:** [arXiv 2606.30573 June 29 2026](https://arxiv.org/abs/2606.30573) — academic
**Date:** June 29, 2026
**Agent level:** n/a (benchmark)
**What:** SWE-Interact reimagines coding benchmarks as user-driven long-horizon sessions where user reveals requirements iteratively. High-performing models scored ~50% on traditional SWE tasks but only ~25% on interactive variant. Top models (Opus 4.8, GPT-5.5) showed better persistence with ambiguous initial instructions. Even leading models suffer from "over-agentic coding, forgetting requirements and technical mistakes." Interactive goal discovery and iterative refinement are a separate capability axis from single-turn benchmarks.
**Evidence level:** Level 1 (academic paper; benchmark methodology not yet independently replicated)
**Key claims:**
- ~50% → ~25% capability drop when requirements revealed iteratively
- "Over-agentic coding, forgetting requirements" = specific named failure mode
- Interactive collaboration is a distinct capability, not just single-turn quality
- Implication: single-turn benchmarks overstate agent readiness for real multi-turn work
