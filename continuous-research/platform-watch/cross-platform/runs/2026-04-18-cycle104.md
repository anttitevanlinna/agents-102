### Finding 1: CIO.com / Scott Bickley — Headless 360 Enterprise Readiness Gaps (Level 2)

**Source:** https://www.cio.com/article/4159536/salesforce-launches-headless-360-to-support-agent-first-enterprise-workflows.html — [domain trade publication] — April 2026
**Date:** April 2026
**Agent level:** Company
**Evidence level:** Level 2 (independent analyst, named, named firm — Info-Tech Research Group; specific verifiable claims)

**What:** Scott Bickley, Info-Tech Research Group, provides the first independent enterprise readiness assessment of Headless 360 post-TDX. Four named gaps:

1. **Pricing opacity:** "There is no mention of cost or the underlying licensing model for this 'headless' experience." Salesforce's pattern is to announce capabilities that require new SKUs — CIOs should "inquire about pricing before establishing architectural dependencies."
2. **Missing SLAs:** No service level agreements for MCP tool calls — "matter materially for real-time agent workflows."
3. **Governance tool immaturity:** Testing Center, Custom Scoring Evals, Session Tracing, A/B Testing API all in "early release" stage. Recommendation: "supplement these tools with their own evaluation frameworks for the next 12-18 months."
4. **Development complexity increase:** Agentforce Experience Layer "adds yet another layer to contend with in the development process for what is already considered a painful development experience."

**Recommendation:** Enterprises should demand "extended pilot and sandbox free of charge to validate the maturity level and ease of use" before committing.

**Positive finding:** MCP tools enabling Claude Code and Codex to access Salesforce directly addresses a historical developer friction point — external IDE access is a genuine improvement.

**Key claims:** Headless 360 is architecturally significant but enterprise-unready on pricing, SLAs, and governance tooling. This is independent confirmation of the "announcement-to-deployment gap" pattern (Pattern 31) for Salesforce's new architectural layer.

### Finding 3: Futurum — Copilot Cowork Enterprise Readiness Assessment (Level 1-2)

**Source:** https://futurumgroup.com/insights/will-ms-copilot-cowork-enable-real-enterprise-ai-collaboration/ — [domain trade publication / analyst] — April 2026
**Date:** April 2026
**Agent level:** Team → Company
**Evidence level:** Level 1 (independent analyst assessment) + Level 2 fragment (Capital Group SVP endorsement)

**What:** Futurum Group's independent enterprise readiness assessment of Copilot Cowork. Key findings:

1. **"Proof point does not yet exist at scale"** for team-level AI coordination producing better outcomes than individuals using separate Copilot sessions independently.
2. **Governance gap:** Microsoft must "publish a clear permission and audit model for Copilot Cowork before it exits Frontier" — without this, IT leaders hesitate beyond experimental deployment.
3. **Accountability problem:** When agent contributes to collaborative output, "who owns the output?" becomes a compliance question in regulated industries. Liability chain ambiguity blocks adoption.
4. **Four watchpoints:** (a) governance specs publication, (b) Google's competitive response at Cloud Next, (c) enterprise pilot-to-production conversion rates, (d) regulated industry formal authorization for shared agent participation.
5. **Context:** 40.2% of CIOs already use Copilot — architectural advantage exists but doesn't translate to Cowork adoption without demonstrated value.

**Capital Group fragment (Level 2):** Barton Warner, SVP Enterprise Technology, cited appreciation for Cowork's task coordination capability "while staying within security boundaries" — single vendor-cited quote, not an independent deployment report.

**Assessment:** Adds independent analytical weight to the existing Cowork deployment gap. The "proof point does not exist" framing from an analyst (not a vendor) is stronger signal than absence of practitioner reviews alone.

### Finding 4: SAMexpert — Agent 365 Hidden Cost Structure Problem (Level 1)

**Source:** https://samexpert.com/agent-365/ — [domain trade publication, licensing specialist] — April 2026
**Date:** April 2026
**Agent level:** Company
**Evidence level:** Level 1 (independent licensing expert, named firm, specific verifiable analysis)

**What:** SAMexpert (the firm that previously identified E7 effective cost >$200/user) publishes the first licensing-specialist breakdown of Agent 365 pre-GA:

**The two-bill problem:** "The same agent appears on two separate invoices: Agent 365 governance on the M365 bill (per-seat, $15/user) and execution costs on the Azure bill (consumption-based)." Different teams manage these separately — no cross-visibility by default. The $15/user is a "governance floor, not the full cost of deploying AI agents."

**Four unresolved gaps at GA (May 1):**
1. No published consumption cost guidance from Microsoft
2. Exact boundary triggering paid licensing not defined
3. No Product Terms documentation available
4. Autonomous agents with independent identities: still in preview at GA

**Adoption prediction:** Microsoft's 3% Copilot adoption rate among M365 subscribers suggests Agent 365 will face similar challenges. SAMexpert anticipates "significant discounting."

**Security caveat:** Threat detection features remain in public preview at launch — creating incomplete protection initially.

**Key claims:** Agent 365 GA launch (May 1) will be structurally incomplete — no consumption cost guidance, no Product Terms, autonomous agent identities in preview. The governance capability is real but enterprises face a pricing opacity problem before they can model actual deployment cost.
