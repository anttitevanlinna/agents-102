#### Finding A3: Academic Research Names the Multi-Agent Memory Gap — arXiv 2603.17787
**Source:** https://arxiv.org/abs/2603.17787 — "Governed Memory: A Production Architecture for Multi-Agent Workflows" — [academic/research] (Personize.ai, industry-backed)
**Date:** March 18, 2026
**Agent level:** Enterprise workflow (agent-to-agent, NOT human-team)
**What:** First academic paper to name the team/org memory gap in enterprise AI deployments: five named problems — memory silos across workflows, governance fragmentation, unstructured memories, redundant context delivery, silent quality degradation. Reports 99.6% fact recall and 92% governance routing precision for their production system. IMPORTANT CAVEAT: This addresses agent-to-agent memory coordination in enterprise workflows, NOT the human-team context sharing gap. Still, formally naming the problem is a signal.
**Evidence level:** Level 2 (single vendor's production system, no independent verification)
**Key claims:**
- Five enterprise memory problems formally named (memory silos, governance fragmentation, unstructured memories, redundant context, silent quality degradation)
- Proprietary architecture claims 99.6% fact recall
- Scope: agent pipelines, not human teams

**A second arXiv paper** (2603.10062, March 2026) identifies "cache sharing across agents" and "structured memory access control" as the two most pressing unsolved challenges. Both papers confirm the problem exists — neither solves the human-team layer.

### Track B: AI-Native Team Leadership

#### Finding B1: Karpathy autoresearch — program.md Is Individual, Not Team-Level
**Source:** https://github.com/karpathy/autoresearch — [practitioner direct]
**Date:** March 7, 2026
**Agent level:** Personal (individual researcher → AI agents)
**What:** autoresearch README clarifies scope: "One GPU, one file, one metric." The program.md pattern is a single researcher directing a single AI agent for overnight experiments. The "massively collaborative" vision remains stated aspiration, not implementation. Shopify CEO Lütke tried it on company data (37 experiments, 19% performance gain overnight) — single executive experiment, not team leadership pattern.
**Evidence level:** Level 2 (named practitioner, specific experiment outcomes)
**Implication for Pattern G:** Strengthens the individual spec-writer dimension. Still doesn't answer how you lead a team of people who are themselves running program.md-style agents.

#### Finding B2: OpenAI Codex Team Operating Model — Minimal Overhead, Full Ownership
**Source:** https://newsletter.eng-leadership.com/p/how-to-build-ai-native-engineering — [practitioner direct, Rasmussen/OpenAI Codex team, Feb 18, 2026] (paywalled, excerpt accessible)
**Date:** February 18, 2026
**Agent level:** Team
**What:** Specific Codex team practices: core contributors <3, full team ~40; engineers own full projects start-to-finish; minimal meetings ("align on things only"); no daily standups, retrospectives, or sprint planning; continuously reinvent how they work "almost weekly" to eliminate bottlenecks; pair work encouraged for exchange. Leadership model: select people + set mission, then get out of the way.
**Evidence level:** Level 2 (practitioner describing own team, partially paywalled)
**Key claims:**
- Team size: core <3, full team ~40 with distributed ownership
- Rituals eliminated: daily standups, retrospectives, sprint planning
- New ritual: weekly reinvention of work practices
- Leadership act: people selection + mission setting, not execution coordination

#### Finding B3: Intercom — Four Specialized Roles Replace Traditional Management Hierarchy
**Source:** https://www.intercom.com/blog/inside-the-ai-first-support-team/ — [practitioner direct, Declan Ivory, Dec 2, 2025]
**Date:** December 2, 2025
**Agent level:** Team (CS function)
**What:** Ivory describes leading an AI-first CS team through four new specialized roles: AI Operations Lead (monitors system performance, surfaces patterns), Knowledge Manager (maintains accuracy of AI's content), Conversation Designer (shapes tone and clarity), Automation Specialist (expands system capabilities). Leadership is system-supervisory: daily monitoring + pattern detection + routing issues to specialists + economic impact reporting. Traditional people-supervision replaced by system-supervision.
**Evidence level:** Level 2 (practitioner describing own team, vendor employee — caveat applies)
**Key claims:**
- Four roles replace traditional CS management hierarchy
- Management act = system supervision + specialist routing
- Daily ritual = monitoring + pattern detection (not standups)

## Cross-Track Connection: Same Gap, Two Levels

The two tracks share a structural root: **the personal level is solved, the collective level is not.**

- Memory: personal memory exists (Zep, Mem0, Letta), team/company memory doesn't
- Leadership: individual spec-writing is understood (Karpathy, Osmani, Mollick), team-level leadership of spec-writers is not

Pattern G's unsolved problem (how do you evaluate a spec-writer?) and Track A's unsolved problem (how do 5 engineers share agent context?) are the same problem from two directions. The team exists at the intersection: a leader writes specs and shares context with a team of people who also write specs and share context. Neither the memory infrastructure nor the evaluation framework exists for this collective layer.

**This is now a Level 3 absence finding:** After 88 cycles, the team/company layer remains empty across memory AND leadership dimensions.

## Orientation

The memory gap is hardening into a research finding: personal memory is a mature market, team/company memory is structurally absent. AWS and Coworker.ai are making company-level claims but neither has independent deployment evidence. The AGENTS.md pattern is the current best practice — better documented in 2026 than before, same fundamental limits.

AI-native team leadership has more new evidence than expected: the Codex team's model (minimal overhead, full ownership, weekly reinvention) is the clearest operational description found. The four-role model from Intercom is the best non-engineering evidence. But the performance evaluation gap — how to assess spec-writer quality — remains completely open after this cycle, with DORA 2025 confirming it's not just a KB gap but a field-wide gap.
