### 4. Copilot DLP Bypass Bug — Level 3 Convergence
**Source:** https://www.theregister.com/2026/02/18/microsoft_copilot_data_loss_prevention/ — [domain trade publication]
**Date:** February 18, 2026
**Agent level:** Personal/Company (security)
**What:** Tracked as CW1226324, detected January 21. Copilot's "work tab" chat read and summarized emails from Sent Items and Drafts carrying confidentiality labels, despite DLP blocks. UK NHS reported it internally as INC46740412. Microsoft deployed fix by late February. Root cause: code defect skipping sensitivity label checks on specific folders. Confirmed by Microsoft, reported by NHS, covered by multiple independent security publications.
**Evidence level:** Level 3 (convergence — vendor confirmation + NHS + BleepingComputer + eSecurity Planet + CyberNews)
**Key claims:** (1) DLP policy bypassed for 3+ weeks (Jan 21 — Feb fix); (2) NHS independently flagged; (3) Root cause = code defect in sensitivity label checking.

### 5. Copilot Studio Security: Tenable Validates Real Attack Vectors
**Source:** https://www.microsoft.com/en-us/security/blog/2026/02/12/copilot-studio-agent-security-top-10-risks-detect-prevent/ — [vendor press release] + https://derkvanderwoude.medium.com/copilot-studio-agent-security-baseline-6df35608ec07 — [practitioner direct]
**Date:** February 12-16, 2026
**Agent level:** Team/Company (security)
**What:** Microsoft's Defender team published top 10 real-world Copilot Studio misconfigurations: unauthenticated agents, maker-credential authentication (agent runs on builder's credentials, not user's), dormant/orphaned agents, email-capable agents vulnerable to prompt injection. Tenable independently validated SSRF attacks via HttpRequestAction and "CoPhish" token-harvesting technique.
**Evidence level:** Level 2 (vendor + independent security researcher validation)
**Key claims:** (1) Unauthenticated agents exist in production; (2) Maker-credential auth = agent uses builder's access; (3) SSRF and CoPhish attacks independently validated by Tenable.

## Orientation

The Microsoft business agent story has three new threads:

1. **Foundry reliability is a real question.** Day 5 post-GA and the only practitioner signals are bug reports and reliability complaints, not success stories. The pre-GA limitations (broken RAG grounding, hidden system prompts) remain unconfirmed as fixed. This matters: if the first adopters can't get stable performance, the 30-90 day adoption window shrinks.

2. **The security surface is expanding faster than governance can keep up.** DLP bypass (Level 3), Copilot Studio misconfigurations validated by Tenable, shadow agents at 29% — and the governance product (Agent 365) doesn't ship until May 1. Microsoft is simultaneously creating the problem and selling the solution.

3. **Copilot Cowork silence is becoming the signal.** At 12+ days post-announcement with zero independent reviews, this is no longer "too early to tell" — it's becoming evidence that either the rollout hasn't happened or early testers have nothing notable to report. Next check: April 1-7 for first Frontier reports.

For the CTO question: Microsoft's position is unchanged — strongest governance vision, weakest deployment evidence, growing security concerns. The Foundry reliability issues add a new dimension: even the developer tier isn't stable yet.
