## What He's Building

Cohen's work spans three interconnected projects, all under the umbrella of "agentic infrastructure":

### 1. Ruflo (formerly Claude Flow) — Agent Swarm Orchestration
- **What:** Multi-agent orchestration platform for Claude Code. Deploys 54+ specialized agents in coordinated swarms with shared memory.
- **GitHub:** [ruvnet/ruflo](https://github.com/ruvnet/ruflo) [practitioner direct]
- **Stars:** ~29,000 | **Forks:** ~3,200 | **Commits:** 6,000+
- **Claims:** ~500,000 downloads, ~100,000 monthly active users across 80+ countries, 250% improvement in effective subscription capacity, 75-80% reduction in token consumption
- **Architecture:** Hierarchical swarms with queen-led coordination. Multiple topologies (mesh, hierarchical, ring, star). Consensus algorithms (Raft, BFT, Gossip, CRDT). 3-scope agent memory (project/local/user). WASM kernels in Rust for policy engine.
- **Memory system:** "RETRIEVE → JUDGE → DISTILL → CONSOLIDATE → ROUTE" learning loop. Successful patterns stored in vector memory. Cross-agent knowledge transfer.

### 2. RuVector — Self-Learning Vector Database
- **What:** Vector database with continuous learning mechanisms. Powers agent memory and the Cognitum chip.
- **GitHub:** [ruvnet/ruvector](https://github.com/ruvnet/ruvector) [practitioner direct]
- **Stars:** ~3,700 | **Forks:** ~447 | **Commits:** 2,559
- **Key differentiator:** Self-learning engine (SONA) that adapts in <1ms through LoRA fine-tuning with Elastic Weight Consolidation. GNN layer learns from query patterns. Claims to learn from every query execution.
- **Architecture:** HNSW indexing with SIMD acceleration, Graph RAG with Leiden community detection, 50+ attention types, local LLM runtime, 230+ SQL functions for PostgreSQL integration.

### 3. Cognitum — Agentic Processing Chip
- **What:** Hardware designed to run always-on AI agents at edge. CES 2026 Innovation Awards Honoree.
- **Claims:** Tens of thousands of agents at ~5 microwatts/inference, <15W total, sub-millisecond response, 500x cheaper than cloud AI.
- **Source:** [Facebook post](https://www.facebook.com/ruvnet/photos/-in-other-news-our-cogntium-v1-agentic-chip-won-an-innovation-award-at-ces-recog/10173508902055430/) [practitioner direct], [LinkedIn](https://www.linkedin.com/posts/reuvencohen_cognitum-v0-the-worlds-first-agentic-processing-activity-7437314135845015553-w9ED) [practitioner direct]
- **Status:** Seed units announced for March 2026 presale. No production deployment evidence found.

### 4. ReasoningBank — Persistent Memory for Agents
- **What:** Self-learning, local-first memory system giving agents persistent knowledge across sessions.
- **Source:** [GitHub issue #811](https://github.com/ruvnet/ruflo/issues/811) [practitioner direct]
- **Philosophy:** "Intelligence without memory isn't intelligence. It's performance art."
- **Architecture:** SAFLA cycle (Store → Embed → Query → Rank → Learn). SQLite-based with 1024-dimensional vectors via SHA-512. Bayesian confidence updates. 12-table schema covering patterns, embeddings, trajectories, collective memory, sessions, neural patterns.
- **Claims:** 2-3ms query latency at 100K+ patterns, 87-95% semantic accuracy, 34% task effectiveness improvement, 73% reduction in bug resolution time.
- **Built on:** Google Cloud AI Research paper "ReasoningBank: Scaling Agent Self-Evolving with Reasoning Memory" (arXiv:2509.25140, Sept 2025).

### 5. SPARC Framework — Agentic Coding Methodology
- **What:** Specification, Pseudocode, Architecture, Refinement, Completion. A deliberate-friction methodology forcing agents to finalize architecture before generating code.
- **Source:** [GitHub](https://github.com/ruvnet/sparc) [practitioner direct], [LinkedIn](https://www.linkedin.com/pulse/automated-code-development-new-sparc-npx-create-sparc-reuven-cohen-8ujwe) [practitioner direct]

His framing is technical/infrastructure-oriented: make agents remember, make them learn from patterns, make them share within swarms. The "organizational" dimension — how this connects to human teams, business processes, and institutional knowledge — is not his focus.

**Connection to our "conditions creator" pattern:** Indirect at best. Cohen builds infrastructure that a conditions creator could theoretically use, but he's not thinking about the role or the organizational dynamics.

## Competing Approaches to Agent Memory (Context)

The AI agent memory space has multiple serious players as of early 2026. Cohen's work exists alongside but is not benchmarked against:

- **Mem0** — Dedicated memory layer, 67% LLM-as-a-Judge score on LOCOMO benchmark, production-focused. [URL](https://mem0.ai/blog/graph-memory-solutions-ai-agents)
- **Letta (formerly MemGPT)** — Memory as first-class agent state, editable memory blocks, stateful runtime. Published benchmarking paper. [URL](https://www.letta.com/blog/benchmarking-ai-agent-memory)
- **Zep** — Episodic and temporal memory, structures interactions into meaningful sequences
- **Hindsight** — Claims 91.4% on LongMemEval vs Mem0's 49.0%
- **MAGMA** — Academic: multi-graph based agentic memory architecture (arXiv:2601.03236)
- **Multi-Agent Memory research** — Academic framing from computer architecture perspective, three-layer memory hierarchy (arXiv:2603.10062)

Cohen's RuVector/ReasoningBank is NOT positioned against these. It exists in a parallel universe of claims without cross-comparison. The established memory frameworks have published benchmarks, peer comparison, and documented limitations. Cohen's work has none of these.

**Short answer: Not yet. Possibly not ever in its current form.**

But the gap between vision and verified execution is large:

Cohen's work is purely technical infrastructure. He's not thinking about teams, roles, or organizational dynamics. The "conditions creator" needs memory infrastructure, yes, but Cohen isn't building for that use case.

4. **Scope is a warning sign.** A single practitioner claiming to build a self-learning vector DB + agent orchestration platform + agentic semiconductor + quantum coherence + genomics processing + 50 attention mechanisms stretches credibility. In the cloud era, his Enomaly was focused. In the AI era, his work is diffuse.
