Platform assessment question: "What moves quickest when the model improves?" Coding agents (Claude Code, Codex) improve at model velocity — no connector layer, no middleware, no enterprise plumbing. Platform agents (Cowork, Agentforce) improve at the speed of the slowest engineering team in the integration chain.

Compounding advantage: coding agents don't just benefit from model improvements — they build the connectors that platform agents are waiting for. An org with coding agent competence builds its own Graph/Entra/Salesforce integrations faster than Microsoft/Salesforce ships them.

Mid-term platform bet: coding agents are the meta-platform. Invest in competence there, build your own plumbing. The improvement cycle is one layer (model), not three (model + connector + compliance).

**Gap:** No framework in our platform assessment explicitly asks "what is the improvement velocity path?" We assess current capability and deployment evidence, but not the structural speed of capability improvement. This is a new dimension for the CTO answer.
