## Findings

### Claude Code: Terminal-first agent with strong reasoning, weak token efficiency
**Source:** https://www.builder.io/blog/codex-vs-claude-code — [practitioner analysis]
**Date:** March 2026
**What:** Steve Sewell (Builder.io CEO) tested Claude Code head-to-head with Codex across real tasks. Found the tools "increasingly similar" in capabilities but architecturally divergent: Claude Code is developer-in-the-loop with deep reasoning, Codex is autonomous/cloud-first. Claude Code uses 3-4x more tokens than Codex for comparable tasks (e.g. Figma task: Claude 6.2M tokens vs Codex 1.5M). Claude produces more documented code with docstrings and edge-case handling; Codex produces more concise code.
**Evidence level:** Level 2 (single practitioner comparative test)

### Claude Code: 79.6% SWE-bench accuracy on $20/mo Pro plan
**Source:** https://hackceleration.com/claude-code-review/ — [practitioner analysis]
**Date:** February 2026
**What:** After Sonnet 4.6 release, Pro plan ($20/mo) users get 79.6% SWE-bench accuracy with 1M context, within 1.2 points of Opus 4.6's 80.8%. This changes the value equation — Gemini CLI (free, 80.6%) and Claude Code Pro ($20, 79.6%) dominate the value quadrant. Paying $100-200/mo for Max gains only 1.2 points of accuracy; the real value is rate limit relief.
**Evidence level:** Level 2 (benchmark analysis)

### Claude Code VS Code marketplace: 5.2M installs, leads Codex
**Source:** https://visualstudiomagazine.com/articles/2026/02/26/claude-code-edges-openais-codex-in-vs-codes-agentic-ai-marketplace-leaderboard.aspx — [domain trade publication]
**Date:** February 2026
**What:** Claude Code for VS Code has 5.2 million installs (rating 4.0/5) vs Codex at 4.9 million installs (rating 3.4/5). Despite being published later, Claude Code leads in both installs and user satisfaction in the VS Code marketplace.
**Evidence level:** Level 3 (marketplace data reflects convergence of many users)

### Usage limits: The #1 practitioner frustration
**Source:** https://www.theregister.com/2026/01/05/claude_devs_usage_limits/ — [general press, bare facts]
**Date:** January 2026
**What:** Multiple developers reported ~60% reduction in token usage allowances. Users hitting limits within 10-15 minutes on Sonnet, calling it "unusable for development work." Anthropic attributed complaints to removal of holiday bonus limits (Dec 25-31 doubled capacity). All Claude product surfaces (claude.ai, Claude Code, Desktop) share the same usage pool. Community-built tools like `ccusage` emerged to track burn rates.
**Evidence level:** Level 3 (convergence — multiple independent developers reporting same issue)

### Claude Agent SDK: Production-capable but operationally difficult
**Source:** https://www.ml6.eu/en/blog/inside-the-claude-agents-sdk-lessons-from-the-ai-engineer-summit — [practitioner analysis]
**Date:** March 2026
**What:** ML6 summarized key AI Engineer Summit lessons on the Agent SDK. Critical insight: harness matters as much as model — same Opus 4.5 scored 78% on CORE benchmark with Claude Code's harness vs 42% with Smolagents. Best practices: use bash as primary tool (not many specialized tools), filesystem-based dynamic context discovery, verification-centered workflows. But production challenges are real: OOM risks with multiple subagents, non-trivial sandboxing, difficult filesystem management across users/teams.
**Evidence level:** Level 2 (summit practitioner reports)

### MCP: De facto standard with 6,400+ registered servers
**Source:** https://en.wikipedia.org/wiki/Model_Context_Protocol — [reference]; https://blog.modelcontextprotocol.io/posts/2026-mcp-roadmap/ — [practitioner direct]
**Date:** February-March 2026
**What:** MCP has become the integration standard for agentic AI. 6,400+ servers on official registry. 97M monthly SDK downloads. Adopted by OpenAI, Google, Microsoft. Donated to Linux Foundation's Agentic AI Foundation (Dec 2025) with OpenAI and Block as co-founders. 2026 roadmap focuses on transport scalability, agent-to-agent communication, and enterprise governance. Streamable HTTP transport enabled remote MCP servers but surfaced scaling gaps (stateful sessions vs load balancers).
**Evidence level:** Level 3 (industry-wide convergence on a single protocol)

### Opus 4.6: Longest autonomous task horizon measured
**Source:** https://moltbook-ai.com/posts/ai-agents-march-2026-roundup — [domain trade publication]
**Date:** March 2026

### Practitioner workflow pattern: Claude for architecture, Codex for execution
**Source:** https://www.builder.io/blog/codex-vs-claude-code — [practitioner analysis]; https://northflank.com/blog/claude-code-vs-openai-codex — [practitioner analysis]
**Date:** March 2026
**What:** Multiple comparison articles independently describe the same workflow: use Claude Code for planning and structural decisions, hand clearly-defined execution tasks to Codex, use Codex review as final check before merging. This aligns with architectural divergence: Claude = "measure twice, cut once" philosophy, Codex = "move fast and iterate." GPT-5.3-Codex leads Terminal-Bench 2.0 at 77.3% vs Claude's 65.4% on terminal-native tasks.
**Evidence level:** Level 3 (convergence — multiple independent sources describe same split)

## What I Looked For But Did Not Find

- **Enterprise team deployment case studies for Claude Code.** No named company reporting "our engineering team of N uses Claude Code in production with these results." All evidence is individual practitioners. The chasm-crossing signal has not appeared.
- **Claude Agent SDK production deployments beyond demos.** Summit examples were impressive demos (ad generation, daily briefs, customer service) but no evidence of sustained production use by a team.
- **Background agent reliability data.** Opus 4.6 METR time horizons are promising but no practitioner reports on real-world background task completion rates.
- **Pricing transparency for agent workloads.** API costs for heavy agentic use (many tool calls, long contexts) remain unclear. The 3-4x token overhead vs Codex is a cost concern with no clear Anthropic guidance.
- **Claude Code in non-coding business domains.** Almost all practitioner reports focus on coding tasks. Evidence for Claude Code/Agent SDK used in finance, HR, compliance, or operations workflows is absent — all enterprise plug-in evidence is vendor announcements (Level 0).

## Orientation

Claude Code is the practitioner favorite for complex reasoning and architecture work, with marketplace data (5.2M VS Code installs) and workflow patterns (Claude for planning, Codex for execution) confirming this position. But the platform has two serious gaps: token efficiency (3-4x worse than Codex) creating unpredictable costs, and usage limits that provoked a user revolt in January 2026. MCP is Anthropic's strongest strategic asset — it has become the industry standard (6,400+ servers, adopted by all major players) and gives Claude Code the richest tool ecosystem. Enterprise distribution is happening through Microsoft and Google partnerships, not Anthropic directly. No enterprise team deployment evidence exists — same gap as Codex.
